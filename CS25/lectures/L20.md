# L20 Generalist Agents in Open-Ended Worlds

This lecture is the agent version of the foundation-model story: build an open world, feed it internet-scale data, and let the agent keep bootstrapping itself.

## Summary
- The talk starts with active kittens versus passive kittens. The point is simple: agents need embodied feedback loops, not just passive observation. Source: `[1:01]` to `[3:15]`.
- Minecraft is the chosen open-ended world because it is procedurally generated, massive, and full of public behavior traces. That makes it a better substrate for generalist agents than a fixed benchmark. Source: `[7:39]` to `[15:12]`.
- MineDojo packages an environment, a knowledge base, and a task suite. MineCLIP then turns video-text pairs into a reward model, giving the system an RLHF-like loop in Minecraft. Source: `[15:21]` to `[20:35]`.
- Voyager turns GPT-4 into a lifelong learning agent using code generation, a skill library, and a curriculum. Eureka applies the same idea to reward-function search for robot dexterity. Source: `[20:43]` to `[37:40]`.
- The forward-looking theme is multimodality. Video, images, audio, and morphology all become part of the interface for embodied agents. Source: `[37:49]` to `[47:40]`.

## Key Concepts
- **Active kitten / passive kitten.** Learning requires action, not just passive exposure.
- **Open-ended world.** Minecraft has no fixed objective, so it is a good stress test for generalist behavior.
- **MineDojo.** An open framework with a task suite, an API, and internet-scale behavior data.
- **MineCLIP.** A video-text contrastive model that acts like a language-prompted reward function.
- **Voyager.** GPT-4 plus Mineflayer plus a skill library plus a curriculum loop.
- **Skill library.** Program summaries become retrieval keys and code becomes the stored skill.
- **Eureka.** GPT-4 proposes reward functions, RL evaluates them, and the best candidates are iterated.
- **R3M / VIP / VPT.** Different ways to extract useful structure from video, reward signals, or pseudo-actions.
- **VIMA / RT-1 / RT-2 / RoboCat.** The lecture treats these as the multimodal, embodied follow-up to the Minecraft stack.

## Non-Obvious Insights
- The scaling axis is not just model size. It is environment richness, memory, and the amount of structured feedback the agent can consume.
- Video alone is not enough for embodiment. You need a bridge from observation to action, or the model will misread what it sees.
- Voyager's skill library matters as much as GPT-4. Memory turns a black-box model into a compounding system.
- The no-gradient Voyager stack works because the world is textifiable. Eureka needs gradients because the robot skill lives in continuous control.
- Internet data is valuable even for robotics because it regularizes representations and improves robustness to visual shifts.
- The cat-on-boxing-video example is the whole problem: passive video is cheap, but it can be deeply misleading without grounding.
