# L03 Vision Transformers: Patchify First, Generalize Later

This lecture starts from a visual question: what counts as a good visual representation, and how do you measure whether a model can adapt from a few examples instead of a giant labeled set?

## Summary
- The lecture defines visual representation learning as few-shot transfer across many task types, and uses VTAB to make that idea measurable. Source: `[0:32]` to `[6:05]`.
- The early scaling story is self-supervised, then semi-supervised, then large-scale supervised pretraining. The big payoff comes when data, model size, and patience all scale together. Source: `[7:15]` to `[15:31]`.
- ViT is intentionally simple: split the image into patches, project them into token embeddings, add a class token and learned position embeddings, then run a BERT-style transformer encoder. Source: `[25:16]` to `[28:23]`.
- On small data ViT loses to ResNet/BiT, but on larger data it catches up and then wins. The model only looks good once scale is real. Source: `[28:37]` to `[30:09]`.
- Learned positional embeddings, patch size, and hybrid convolutional front-ends are the real engineering knobs. Source: `[31:02]` to `[43:33]`.

## Key Concepts
- **General visual representation.** The target is a model that can adapt from a few examples across many visual tasks. Source: `[0:32]` to `[6:05]`.
- **VTAB.** The Visual Task Adaptation Benchmark formalizes that few-shot transfer goal. Source: `[4:05]` to `[5:54]`.
- **Pretraining vs transfer.** The lecture uses upstream/pre-training and downstream/transfer as the core split. Source: `[6:05]` to `[6:49]`.
- **Self-supervised and semi-supervised learning.** These are tried before the large-scale supervised push. Source: `[7:15]` to `[8:38]`.
- **Big Transfer (BiT).** Scaling supervised pretraining on much larger image corpora gives the first large gains. Source: `[10:53]` to `[15:31]`.
- **Patch tokens.** ViT cuts an image into fixed-size patches, flattens them, and linearly projects them like word tokens. Source: `[26:30]` to `[28:23]`.
- **Class token and learned position embeddings.** The transformer reads a special class token, and the position embeddings learn spatial structure on their own. Source: `[27:41]` to `[33:44]`.
- **Patch order invariance.** If patch order changes after training, performance drops because the learned positions no longer mean the same thing. Source: `[31:02]` to `[33:44]`.
- **Hybrid models.** A small ResNet front-end plus a transformer can help when data is scarce. Source: `[35:05]` to `[39:18]`.
- **Patch-size tradeoff.** Smaller patches mean more tokens, more attention work, and worse quartic scaling in image size. Source: `[44:05]` to `[47:54]`.
- **Receptive field drift.** Early attention heads look locally and globally; deeper heads mostly look globally. Source: `[48:01]` to `[49:17]`.

## Non-Obvious Insights
- ViT is not "ResNet with some attention sprinkled on top." The lecture is explicit that the real move is to let the transformer own the whole representation stack. Source: `[25:16]` to `[28:23]`.
- Learned positional embeddings recover neighborhood structure without hard-coded locality. The model discovers adjacency from data. Source: `[31:45]` to `[33:44]`.
- Locality bias is a tradeoff, not a universal good. It helps in low-data settings, but scale can reward more flexible models. Source: `[53:34]` to `[57:58]`.
- Model size in vision is not cleanly comparable to model size in language. Parameter counts and compute tell different stories. Source: `[1:05:35]` to `[1:07:17]`.
- The real story is scale discipline: large datasets, large models, and patience make the transformer slope beat the ResNet slope. Source: `[59:20]` to `[1:05:19]`.
- The lecture keeps showing the same pattern as in language: what looks weak at small scale can become the winning inductive bias once scale is large enough. Source: `[28:37]` to `[30:09]` and `[59:20]` to `[1:05:19]`.
