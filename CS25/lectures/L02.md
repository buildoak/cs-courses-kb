# L02 Transformers in Language: Scale Turns Next-Token Prediction into a General Interface

This lecture asks a blunt question: if next-token prediction is such a local objective, why does it end up carrying reading comprehension, translation, summarization, and code?

## Summary
- Language modeling moves from n-grams to RNNs and LSTMs, then to autoregressive transformers because longer context and better coherence matter. Source: `[0:27]` to `[2:33]`.
- GPT-2 shows that prompt-based completion can already do zero-shot reading comprehension, summarization, and translation. Source: `[15:21]` to `[18:08]`.
- GPT-3 reframes the whole thing as meta-learning over tasks: training builds skills, and inference adapts to the prompt without gradient updates. Source: `[18:16]` to `[21:07]`.
- The real motivation at OpenAI is unsupervised learning from huge unlabeled text corpora, not language modeling as an end in itself. Source: `[8:12]` to `[10:53]`.
- The same recipe later powers code models, because code is plentiful, structured, and easy to score with tests. Source: `[32:36]` to `[48:10]`.

## Key Concepts
- **Autoregressive language modeling.** The model predicts the next token from previous tokens, then uses that same mechanism at inference time. Source: `[1:53]` to `[2:33]`.
- **Prompt/completion interface.** Downstream tasks are converted into text prompts, and the model continues them. Source: `[16:02]` to `[17:02]`.
- **Zero-shot, one-shot, few-shot.** The lecture uses the same task framing to show capability emerging as model size increases. Source: `[16:17]` to `[21:07]`.
- **Neural scaling laws.** The jump from GPT-2 to GPT-3 is discussed as scale following a predictable performance curve. Source: `[5:20]` to `[6:00]`.
- **Analysis by synthesis.** If a model can generate coherent samples, it has probably learned useful structure for understanding too. Source: `[11:34]` to `[12:13]`.
- **Unsupervised learning.** GPT is motivated as a way to learn from unlabeled internet text, where labels are scarce and expensive. Source: `[9:42]` to `[10:53]`.
- **Minimal fine-tuning.** GPT-1 shows the early pattern: train a language model, then add a thin task head. Source: `[14:34]` to `[15:17]`.
- **Meta-learning view of GPT-3.** The model learns over a distribution of tasks and adapts in context at inference time. Source: `[18:20]` to `[19:04]`.
- **Code as language.** The talk closes by showing that code generation follows the same completion logic, then becomes stronger after code-specific fine-tuning. Source: `[32:36]` to `[48:10]`.

## Non-Obvious Insights
- Next-token prediction looks local, but it can still force the model to encode long-range structure when the completion depends on it. Source: `[12:15]` to `[13:00]`.
- "Unsupervised learning" is the better label here. The point is not to model text for its own sake, but to learn transferable structure from unlabeled data. Source: `[8:44]` to `[10:53]`.
- GPT-2 turns language tasks into prompt engineering problems before that term was fashionable. Source: `[15:40]` to `[18:08]`.
- GPT-3 is not just a bigger GPT-2. The important change is that prompting starts to behave like in-context adaptation. Source: `[18:20]` to `[21:07]`.
- The few-shot gap shrinks with scale, which means larger models are not only better, but more prompt-responsive. Source: `[20:45]` to `[21:07]`.
- Code is a good proving ground because the output is structured enough to score automatically, unlike most natural language tasks. Source: `[33:01]` to `[35:52]` and `[45:52]` to `[46:10]`.
