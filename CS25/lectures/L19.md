# L19 Low-Level Embodied Intelligence with Foundation Models

This lecture is the robotics version of the foundation-model story: build a safer world model, scale offline data, and use language as the control surface.

## Summary
- Real-world robotics is hard because the world is messy, physical, and unforgiving. The Coke-can and upside-down-can examples show why a robot needs a world model, not just a policy. Source: `[0:05]` to `[4:06]`.
- Gibson and iGibson turn simulation into a safe playpen. The point is not only navigation, but richer manipulation, articulation, and perception-action loops. Source: `[4:10]` to `[10:18]`.
- The robotics recipe is large offline data, a high-capacity transformer, and language as glue. SayCan, RT-1, and PaLM-E progressively collapse planner, policy, and affordance machinery. Source: `[10:23]` to `[37:52]`.
- RT-2 and Open X Embodiment push the same recipe further. Mixing internet-scale multimodal data with robotics data improves generalization and positive transfer. Source: `[38:06]` to `[57:44]`.
- The deeper interface is reward functions. Language can generate motion descriptions, which a reward coder turns into code and an optimizer turns into robot motion. Source: `[58:06]` to `[1:09:40]`.

## Key Concepts
- **Embodied intelligence.** The talk frames robotics as a core AI problem, not an optional add-on.
- **World models.** Robots need to model geometry, physics, and the consequences of actions.
- **Gibson / iGibson.** Scanned houses become interactable environments with more degrees of freedom than plain navigation benchmarks.
- **Moravec's paradox.** Reasoning is cheap compared with sensorimotor control.
- **SayCan.** Language-model scores are filtered by affordance scores so the planner only proposes executable actions.
- **RT-1.** Vision goes through a tokenized backbone, actions are discretized into bins, and a transformer maps tokens to robot control.
- **PaLM-E.** Multimodal tokens from vision or robot sensors are mapped into the text embedding space so one model can plan and ground.
- **RT-2.** Internet-scale pretraining plus robotics data improves generalization to unseen objects, symbols, and instructions.
- **Open X Embodiment / RT-X.** Pooling robot data across labs increases skill diversity and makes joint training more useful.
- **Language to reward.** Motion descriptions become reward code, then RL or MPC uses that reward to produce motion.

## Non-Obvious Insights
- The bottleneck is not just model capacity. It is the action interface. Robotics data is tiny, expensive, and badly aligned with language pretraining.
- A robot policy can generalize by reusing semantic knowledge, but only if the output format is compatible with the pretrained model.
- RT-2's gains come from both scale and retention. The internet mixture acts as a regularizer, not just extra data.
- High-level planning can be taught with surprisingly few demonstrations once low-level skills exist.
- Reward functions are a stronger abstraction than raw actions because they transfer across tasks and let the optimizer handle dexterity.
- Safety is a first-class systems problem. The lecture explicitly points to catastrophic failures if the prompt is misunderstood.
