# L04 Decision Transformer: Return Conditioning as Control

This lecture treats offline RL as sequence modeling: feed trajectories in, predict actions out, and use returns-to-go as the control signal.

## Summary
- The speaker frames transformers as a good fit for RL because they scale cleanly, train stably, and already dominate perception work; classic deep RL is much smaller and much noisier to train. Source: `[0:05]` to `[7:30]`.
- Decision Transformer turns each offline trajectory into tokens for state, action, and returns-to-go, then predicts the next action causally from past context. Source: `[11:14]` to `[16:44]`.
- The model is intentionally non-Markovian. Longer context helps partial observability, and the return token acts as a control knob for desired behavior instead of a fixed label. Source: `[19:17]` to `[21:26]`, `[52:13]` to `[52:59]`.
- On Atari, OpenAI Gym, and a sparse-reward Key-To-Door environment, Decision Transformer is competitive with strong offline RL baselines and does especially well when long-horizon credit assignment matters. Source: `[32:46]` to `[34:06]`, `[53:48]` to `[55:06]`, `[59:01]` to `[60:23]`.
- Return conditioning also works as a weak multitask interface, and the same causal machinery can be reused to predict rewards or returns-to-go instead of actions. Source: `[37:07]` to `[39:10]`, `[1:00:00]` to `[1:03:20]`.

## Key Concepts
- **Offline RL.** Learn from logged trajectories without further environment interaction. Source: `[4:27]` to `[5:37]`.
- **Causal transformer.** Read trajectories left to right and predict actions from the prefix. Source: `[11:43]` to `[12:56]`.
- **Returns-to-go.** Sum of future rewards from the current timestep until episode end; this is the conditioning token that sets the target behavior. Source: `[13:03]` to `[14:36]`.
- **Trajectory tokenization.** Each timestep is encoded as state, action, and return tokens. Source: `[15:21]` to `[16:17]`.
- **Non-Markov conditioning.** The model keeps history on purpose so it can use past context under partial observability. Source: `[19:03]` to `[20:35]`.
- **Context length `K`.** The lookback window is a key hyperparameter and longer context materially helps. Source: `[15:01]` to `[15:17]`, `[52:13]` to `[52:59]`.
- **Percent BC.** Filter the top-return trajectories and do behavior cloning on that subset; this is a much stronger baseline than naive BC. Source: `[45:03]` to `[48:17]`.
- **CQL.** The talk uses conservative Q-learning as the main TD-learning baseline for comparison. Source: `[33:10]` to `[33:21]`, `[49:39]` to `[49:56]`.
- **Key-To-Door.** A sparse-reward, long-credit-assignment benchmark used to stress the method. Source: `[59:01]` to `[1:00:23]`.

## Non-Obvious Insights
- The return token is not a reward leak. It is an intent signal: at test time you ask for a target return, and the model tries to generate a matching rollout. Source: `[37:07]` to `[39:10]`.
- The method is closer to stitched behavior than memorized trajectories. Good subsequences from suboptimal rollouts can be recombined into stronger behavior. Source: `[50:29]` to `[51:58]`.
- Sparse rewards are not the weak point people might expect. DT degrades only mildly when rewards are delayed, while CQL drops sharply. Source: `[53:48]` to `[55:06]`.
- Return conditioning is a primitive multitask interface. Different target returns induce different policies without changing weights. Source: `[42:15]` to `[44:25]`.
- The critic story comes for free once the architecture exists. Swap the output target from actions to rewards and the same transformer starts behaving like a long-horizon evaluator. Source: `[1:00:00]` to `[1:03:20]`.
