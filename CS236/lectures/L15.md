# L15 Evaluation of Generative Models

Lecture 15 says the quiet part out loud: evaluation is not a single number. If you care about compression, likelihood is fine. If you care about sample quality, feature distance and human judgment start to matter. If you care about downstream use, evaluation becomes task-specific. Source: transcript `[0:05]` to `[1:21:25]`.

## Summary
- The lecture frames evaluation around the use case: density estimation, compression, sample quality, representation learning, or prompting. Source: transcript `[6:14]` to `[8:57]`.
- Likelihood is the right metric when the model exposes tractable density and you actually care about compression or probability estimation. Source: transcript `[9:12]` to `[13:01]`.
- Inception Score measures classifier sharpness and label diversity, but it can hide within-class mode collapse. Source: transcript `[40:31]` to `[45:26]`.
- FID and KID both compare generated and real samples in feature space, but they trade off theory, compute, and interpretability differently. Source: transcript `[47:31]` to `[56:27]`.
- HEIM widens the metric set for text-to-image models to include alignment, robustness, aesthetics, originality, and bias; downstream task performance remains the cleanest test for learned features. Source: transcript `[56:32]` to `[1:00:36]`.

## Key Concepts
- **Likelihood / compression.** Maximum likelihood is a reasonable objective only when the thing you care about is density estimation or code length. Source: transcript `[9:12]` to `[13:07]`.
- **Inception Score.** A classifier-based metric that rewards both sharp predictions and high marginal label diversity. Source: transcript `[40:31]` to `[45:26]`.
- **FID.** Fit Gaussians to features extracted by a pre-trained network and compare them with the Wasserstein-2 distance. Source: transcript `[47:31]` to `[53:03]`.
- **KID / MMD.** Compare feature distributions with a kernel two-sample test instead of a Gaussian approximation. Source: transcript `[53:03]` to `[56:27]`.
- **Human evaluation.** Direct, but expensive and noisy, so it does not scale cleanly. Source: transcript `[35:33]` to `[39:13]`.
- **HEIM.** A broader benchmark for text-to-image quality, robustness, originality, and bias. Source: transcript `[57:30]` to `[58:47]`.
- **Downstream tasks.** If you care about features, the cleanest metric is performance on the task you actually want to solve. Source: transcript `[59:07]` to `[1:00:04]`.

## Non-Obvious Insights
- Inference: compression is a proxy, not a universal truth. It weights every bit equally, which is exactly why it can miss the bits you actually care about.
- Inference: Inception Score can look good even when the model is collapsing within a class, because it never directly compares generated samples to real ones.
- Inference: FID became popular because it is computable and feature-aware, not because it is mathematically perfect.
- Inference: evaluation turns into task design as soon as you move from "make samples" to "use the model for something."

## Connections
- To L13 and L14: diffusion models can train stably and sample well, but the lecture shows that those properties still need a metric that matches the actual downstream use.
- To the course project: if the project is text-to-image or image generation, the right evaluation bundle is usually alignment + quality + bias + originality, not a single scalar.
- To language models: perplexity is just likelihood with a different name, but prompting and downstream task success are the real deployment questions.
- To representation learning: if the goal is features rather than samples, the evaluation should move to the downstream classifier or clustering task.

## Referenced Papers
- **Improved Techniques for Training GANs.** Salimans, Goodfellow, Zaremba, Cheung, Radford, Chen, 2016. The paper behind Inception Score. `https://arxiv.org/abs/1606.03498`
- **GANs Trained by a Two Time-Scale Update Rule Converge to a Local Nash Equilibrium.** Heusel, Ramsauer, Unterthiner, Nessler, Hochreiter, 2017. The paper that popularized FID. `https://arxiv.org/abs/1706.08500`
- **Demystifying MMD GANs.** Bińkowski, Sutherland, Arbel, Gretton, 2018. The KID / MMD feature-space comparison line. `https://arxiv.org/abs/1801.01401`
- **Holistic Evaluation of Text-To-Image Models.** Lee et al., 2023. The HEIM benchmark from Stanford. `https://arxiv.org/abs/2311.04287`
