# L08: Data-Parallel Thinking

## Summary
- If you treat data as sequences, `map`, `fold`, and `scan` become the basic algebra of parallel programming.
- `map` is parallel by construction; each element is independent once the function only sees one item at a time.
- `fold` is only safely parallel when the combiner is associative, which is why `plus` works and arbitrary functions do not.
- The work-efficient scan algorithm uses upsweep / downsweep, but the fastest implementation depends on the machine shape.
- Segmented scan handles ragged collections and turns sparse matrix and graph-style workloads into regular data-parallel operators.

## Key Concepts
- **Sequence**: an ordered collection with restricted access, used here instead of raw arrays.
- **Map**: apply a unary function to every element and produce a new sequence.
- **Fold**: reduce a sequence to a single value using an associative combiner.
- **Scan**: compute all prefix results instead of just the final reduction.
- **Inclusive vs exclusive scan**: inclusive includes the current element; exclusive stops before it.
- **Work vs span**: total work and critical-path length can point to different "best" algorithms.
- **Upsweep / downsweep**: the tree phases of the work-efficient scan algorithm.
- **Segmented scan**: scan over a sequence of sequences, or a flat array plus boundary flags.
- **Gather / scatter**: the data-movement primitives for pulling data in or pushing it out by index.
- **CSR**: compressed sparse row, the canonical sequence-of-sequences representation for sparse matrices.

## Non-Obvious Insights
- The best scan algorithm is machine-dependent. The work-efficient version is not automatically faster on SIMD hardware.
- On a warp, the `n log n` version can beat the `O(n)` version because it keeps lanes busy and avoids idle lanes in the tree collapse.
- Ragged problems become regular if you separate payload from boundaries. That is what segmented scan is really doing.
- Sparse matrix multiplication is not a special-case algorithm here. It is map + gather + segmented scan.
- The library boundary matters. Once an operation is expressed as a sequence primitive, the runtime can fuse, reorder, or specialize it.
- The lecture's point is not "use these operators because they are elegant." It is "use them because they expose parallel structure without exposing dependencies."
