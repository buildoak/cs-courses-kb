# L05: Performance Optimization I: Work Distribution and Scheduling

## Summary
- Start with the simplest assignment scheme. If the workload is predictable, static chunking usually wins on overhead.
- When task cost is unknown or heavily skewed, dynamic assignment through a shared counter or queue can recover balance, but only if the tasks are large enough to pay for synchronization.
- Granularity is the knob. Smaller chunks balance better; larger chunks amortize locks and atomics.
- Recursive divide-and-conquer reveals parallelism naturally. Cilk models that with `spawn` and `sync`, and the runtime is free to schedule it however it wants.
- Work stealing is the answer once tasks are recursive. Local threads push and pop from the bottom, thieves steal big work from the top, and greedy join scheduling keeps locality and balance.

## Non-Obvious Insights
- A dynamic scheduler can lose to a static one when the work units are too small.
- Better balance is not free. Synchronization overhead is the tax you pay for flexibility.
- Recursive work stealing works because the expensive work stays near the top, so thieves grab large chunks instead of causing constant churn.
- Random victim selection is good enough in practice and has clean theoretical guarantees.
- `sync` is mostly bookkeeping on stolen work blocks. If nothing was stolen, there is nothing to wait for.
