# L11: Cache Coherence

## Summary
- The lecture starts with a Spark recap: RDD fusion, narrow dependencies, and lineage-based fault tolerance are the locality story in distributed form.
- Cache coherence is the shared-memory version of the same problem: keep one address ordered across many private caches.
- The core guarantee is single-writer, multiple-reader, plus a data-value invariant that preserves the most recent write.
- Snooping coherence uses a bus for two reasons only: serialization and broadcast.
- MSI is the first concrete write-back invalidate protocol: Invalid, Shared, Modified, with bus read, bus read exclusive, and bus write-back transactions.
- Shared caches and write-through protocols are too bandwidth-hungry to scale cleanly.

## Key Concepts
- **RDD lineage**: Spark keeps a replayable log of transformations instead of storing every intermediate result.
- **Fusion**: narrow dependencies let Spark collapse work into one pass, just like a compiler would.
- **Cache line**: coherence operates on lines, not individual bytes.
- **Tag / dirty bit / state**: cache metadata tracks what line is present, whether it is dirty, and who owns it.
- **Single-writer, multiple-reader**: at any time, one cache may modify a line while many may read it.
- **Data value invariant**: the value visible in the read-only epoch must be the one last written in the read-write epoch.
- **Snooping**: every cache watches the shared bus for coherence traffic.
- **Bus serialization**: one transaction at a time creates a global order for coherence actions.
- **Write-through vs write-back**: write-through is simple but floods the interconnect; write-back needs protocol state.
- **MSI**: the three-state protocol introduced at the end of the lecture.

## Non-Obvious Insights
- The Spark detour is not a detour. It is the same locality/fusion problem wearing a distributed-systems mask.
- Locks do not solve coherence. Synchronization and cache coherence are different layers.
- A bus is valuable not because it is elegant, but because it gives serialization and broadcast for free.
- Shared caches can reduce misses, but they also create interference and capacity conflicts between threads.
- Directory-based coherence is about scaling the interconnect, not changing the coherence contract itself.

## Connections
- This is the hardware prerequisite for shared-memory parallel programming.

- The bus, crossbar, and directory discussion is really an interconnection-network problem: correctness is local, scalability is not.
