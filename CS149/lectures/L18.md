# L18: Hardware Specialization

## Summary
- Energy efficiency is the forcing function: Dennard scaling ended, so performance now comes from specialization.
- The lecture walks the specialization ladder: Anton for molecular dynamics, TPU for dense matrix multiply, and FPGAs as the programmable middle ground.
- Spatial turns accelerator design into a programming model with explicit memory hierarchy, data movement, and parallelism control.
- The main performance tools are tiling, SIMD reduction, pipelining, and double buffering.
- Streaming execution with FIFOs makes attention-style workloads cheaper by avoiding huge intermediate matrices.

## Key Concepts
- **Energy = power x time**: fixed power budgets make efficiency the dominant constraint.
- **Specialization**: design hardware around the workload instead of forcing the workload onto a general-purpose core.
- **Anton**: a specialized accelerator for molecular dynamics and n-body style simulation.
- **TPU**: built to make dense matrix multiply fast, because that is the core kernel in many ML workloads.
- **FPGAs**: configurable logic blocks plus hard macroblocks for memory and multiplication.
- **Sparse data**: the hardware/model can turn sparse inputs into dense working buffers for efficient processing.
- **Spatial language**: explicit memory templates, `Accel` blocks, `for each`, `reduce`, and explicit scheduling.
- **FIFOs**: the streaming bridge between pipeline stages.
- **Tiling**: break computation and memory into manageable blocks.
- **SIMD reduction**: use parallel multiplies plus a reduction tree for the inner loop.
- **Pipelining**: overlap independent stages, but pay with extra buffering.
- **FlashAttention**: reorder attention to reduce materialization and FIFO depth.
- **Compiler responsibility**: banking and buffering are handled by the compiler once the algorithm and memory hierarchy are exposed.

## Non-Obvious Insights
- Sparse data and streaming are both memory problems first, compute problems second.
- Pipelining is close to free performance, but only if you pay for the extra storage it needs.
- The programming model is the real shift: the programmer specifies hardware-facing structure, not just kernels.
- FlashAttention is not just an ML trick. It is a memory-footprint reduction strategy that fits the streaming model.
- Spatial is closer to a hardware specification language than to CUDA-style kernel programming.
- The big gains come from matching the algorithm’s locality and parallel structure to the accelerator, not from brute force.

## Connections
- This is the hardware-specialization end of the course’s parallelism and locality story.
