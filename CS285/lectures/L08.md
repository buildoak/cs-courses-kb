# L08 Deep Q-Learning

This lecture makes Q-learning practical by separating three things that should not move at the same speed: experience collection, target construction, and parameter updates.

## Key Concepts
- **Replay buffer.** Storage that turns correlated online experience into shuffled minibatches.
- **Moving target.** The bootstrap label changes as the network changes.
- **Target network.** A lagged copy of `Q` used to stabilize target computation.
- **DQN.** Deep Q-learning with replay buffers and target networks.
- **Double Q-learning.** Separate action selection from value evaluation to reduce overestimation.
- **N-step return.** A multi-step target with a variance/bias trade-off.
- **Huber loss.** A robust alternative to squared error for large Bellman residuals.
- **Polyak averaging.** A soft update rule for target networks.
- **Epsilon-greedy.** The default exploration rule for collecting diverse experience.
- **NAF.** A quadratic `Q` parameterization that makes the continuous-action `argmax` closed form.
- **DDPG.** A continuous-control variant that learns an approximate maximizer.

## Non-Obvious Insights
- Replay buffers are not just storage. They reshape the data distribution enough to make the minibatch update behave like supervised learning again.
- Target networks and replay buffers solve different failures. One breaks correlation; the other slows the target.
- Double Q-learning is a selection/evaluation split, not a new objective. It attacks overestimation by decorrelating the `argmax` from the value estimate.
- `n`-step returns can speed up early learning, but they stop being cleanly off-policy once `n > 1`.
- The continuous-action problem is mostly an argmax problem. The Bellman recursion stays the same; the hard part is finding the best action fast enough.
- Absolute Q-values can be badly calibrated even when the ordering is useful. The policy can still improve off the relative values.
