# L07 Value Function Methods

This lecture turns policy iteration into a sampled, model-free loop. Evaluate the current policy, improve it with `argmax`, then collapse the whole thing into fitted value iteration and fitted Q iteration.

## Summary
- Policy iteration is the same two-step loop throughout: estimate the advantage of the current policy, then build a greedy implicit policy by taking `argmax` over that advantage. Source: `/tmp/agent-mux-501/01KN84E8K6FVCSWAN2FK2QBBPJ/research-116/L07/01.txt` `[0:01]` to `[5:32]`.
- In tabular MDPs with known transitions, dynamic programming can compute `V^pi` exactly, and value iteration converges because the Bellman operator is a contraction in the infinity norm. Source: `/tmp/agent-mux-501/01KN84E8K6FVCSWAN2FK2QBBPJ/research-116/L07/04.txt` `[0:01]` to `[6:38]`.
- Replacing exact expectations with sampled `s, a, s'` tuples gives fitted value iteration and fitted Q iteration, which no longer require the transition model. Source: `/tmp/agent-mux-501/01KN84E8K6FVCSWAN2FK2QBBPJ/research-116/L07/02.txt` `[0:00]` to `[9:16]`.
- The Q-function version keeps the policy implicit: the max over `Q` is the greedy policy, so policy improvement is hidden inside the backup instead of being represented by a separate network. Source: `/tmp/agent-mux-501/01KN84E8K6FVCSWAN2FK2QBBPJ/research-116/L07/02.txt` `[5:04]` to `[11:12]`.
- Q-learning is the online special case of fitted Q iteration; epsilon-greedy collection and repeated updates make the loop practical once you give up on exact dynamics. Source: `/tmp/agent-mux-501/01KN84E8K6FVCSWAN2FK2QBBPJ/research-116/L07/03.txt` `[0:00]` to `[11:57]`.

## Key Concepts
- **Policy iteration.** Alternate between evaluating a policy and improving it.
- **Bellman operator.** The nonlinear backup whose fixed point is `V*`.
- **Tabular MDP.** Finite state/action space where exact dynamic programming is possible.
- **Value iteration.** Repeated application of the Bellman operator until convergence.
- **Fitted value iteration.** Approximate value iteration with supervised regression on max targets.
- **Fitted Q iteration.** The Q-function version of fitted value iteration.
- **Implicit policy.** The greedy policy can live inside the `argmax` over `Q`.
- **Q-learning.** The online bootstrap update that becomes the practical model-free variant.
- **Epsilon-greedy exploration.** The simple data-collection rule that keeps the state space from collapsing too early.

## Non-Obvious Insights
- The `max` in value iteration is not a cosmetic detail. It is the policy-improvement step hiding inside the Bellman backup.
- Once you represent the policy implicitly as an `argmax` over `Q`, you no longer need an explicit policy object to keep iterating.
- The sampled `s, a, s'` form is the real unlock for model-free RL: because `Q(s, a)` conditions on the action directly, the same dataset stays useful as the policy changes.
- The Bellman contraction proof is why tabular value iteration is safe. Later neural Q-learning keeps the same update shape but loses the guarantee.
- Fitted Q iteration and Q-learning are the same idea at different speeds: one is batchy and conservative, the other is online and noisy.
