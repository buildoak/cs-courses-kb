## Summary
- Value-based methods throw away the explicit policy network and recover the policy implicitly with an argmax over value or Q estimates.
- Policy iteration becomes dynamic programming when the transition model is known. Fitted value iteration is the model-free version that projects Bellman backups onto a function class.
- The curse of dimensionality is the real reason the lecture leaves tables behind. Image-sized state spaces make tabular value functions impossible.
- Fitted Q-iteration is the Q-value version of the same loop, and online Q-learning is the one-sample special case.
- The big warning label is that nonlinear function approximation breaks the clean contraction story. Tabular value iteration converges; fitted value iteration need not.

## Key Concepts
- **Policy iteration.** Alternate between evaluating the current policy and improving it with a greedy argmax.
- **Value iteration.** Repeated Bellman backups on a value function.
- **Bellman operator.** The fixed-point map whose contraction gives tabular convergence.
- **Projection operator.** The L2 regression step that snaps a Bellman backup back into the representable function class.
- **Fitted value iteration.** `V <- Pi B V`, the projected Bellman update.
- **Fitted Q-iteration.** `Q <- Pi B Q`, the Q-function version of the same idea.
- **Implicit policy.** The policy is not represented directly; it is decoded from the value function.
- **Off-policy sample reuse.** Value methods can train from older transitions because the data lives at the state-action level.
- **Epsilon-greedy / Boltzmann exploration.** Simple action-selection rules that feed the replay pool without changing the backup rule.
- **Curse of dimensionality.** The table explodes with state dimension, so approximation becomes mandatory.

## Non-Obvious Insights
- The policy is the argmax, not the thing you store. That is why value-based methods are conceptually closer to decoding than to policy learning.
- The Bellman backup and the projection are both harmless in isolation. Their composition can still fail because they contract in different norms.
- The lecture's key instability point is geometric, not cosmetic. `B` contracts in the infinity norm; `Pi` projects in L2; `Pi B` is not guaranteed to contract in either.
- Value-based methods are tempting because they reuse old data. The price is that the convergence story gets weaker once you leave the tabular case.
- Q-learning only looks like gradient descent. The target moves, so the update is not the gradient of a fixed objective.
- The lecture keeps showing the same motif: sample, estimate return, improve policy. Value methods just move the policy improvement into the argmax.

## Referenced Papers
- Sutton & Barto, *Reinforcement Learning: An Introduction* - the standard textbook for policy iteration, Bellman backups, and Q-learning.
- Bertsekas, *Dynamic Programming and Optimal Control* - the classical fixed-point and contraction treatment behind value iteration.
- Puterman, *Markov Decision Processes: Discrete Stochastic Dynamic Programming* - the MDP reference for policy/value iteration.
- Powell, *Approximate Dynamic Programming* - the canonical approximation-and-projection reference for fitted value methods.
- Watkins, *Learning from Delayed Rewards* - the original Q-learning formulation.
- Riedmiller, *Neural Fitted Q-Iteration* - the batch-mode Q-learning variant with function approximation.
