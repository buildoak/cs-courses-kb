# CS230: Deep Learning

Stanford's flipped deep learning course, pinned here to the 2025 offering. The official syllabus is Fall 2025, while the public playlist titles the lecture run Autumn 2025. The arc starts with neural nets and project fundamentals, then pushes into robustness, generative models, RL, LLM applications, and model introspection.

## Syllabus

### Core Arc

The official 2025 syllabus moves through:

- Class introduction and course logistics
- Key AI concepts through case studies
- Full cycle of a deep learning project
- Adversarial robustness and generative models
- Deep reinforcement learning
- Career advice and reading research papers
- Democracy Day / no class
- Beyond the model: enhancing LLM applications
- Career advice plus guest lecture material
- What is going on inside my model? and class wrap-up

### Assessment

- **Final project, 40%**: proposal, milestone, final report, poster session
- **Midterm, 25%**
- **Programming assignments, 25%**
- **Quizzes, 8%**
- **Meeting attendance, 2%**

### Course Mechanics

- The course uses a flipped format.
- In-person Tuesday lectures are paired with Coursera modules completed at home.
- Friday TA sections add hands-on guidance and project support.
- Project teams are 1-3 students.
- The syllabus says assignments are due on Tuesdays at 11:00 a.m. PST, 30 minutes before lecture.
- The project page frames the class project as something that should be motivating and technically challenging, with application, algorithmic, or theoretical directions.
- The project rubric values technical quality, significance, and novelty.
- The project page explicitly says strong CS230 projects can be publishable or nearly publishable.

## Why It Matters

- It is the clean public bridge from deep learning theory to real project execution.
- It does not stop at model mechanics. The syllabus reaches project design, robustness, generative modeling, RL, LLM applications, and model interpretability.
- The flipped format keeps fundamentals at home and uses live class time for advanced topics, guest lectures, and project strategy.
- The project structure forces the hard part of deep learning: picking a problem, getting data into shape, iterating, evaluating, and writing up results.
- The grading rubric rewards technical quality, significance, and novelty, so the class pushes students away from toy demos and toward real work.
- The project guide explicitly treats publishable or nearly publishable work as a valid target. That changes the gear ratio.
