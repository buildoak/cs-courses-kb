# CS230 L07: Beyond LLM

This lecture is the vertical axis of LLM application design: prompting, retrieval, workflow decomposition, evals, and multi-agent systems. Source: transcript `[0:11]` to `[1:49:45]`.

## Summary
- The lecture frames the goal as boosting a base model with prompt design, chains, RAG, agentic workflows, and evals instead of rushing to fine tuning. Source: transcript `[1:37]` to `[3:31]`, `[18:11]` to `[53:23]`.
- Prompt engineering is treated as a real skill, not a slogan: better instructions, examples, role prompts, reflection, chain-of-thought, and prompt chaining all move the frontier. Source: transcript `[18:14]` to `[41:12]`.
- Fine tuning is positioned as expensive, slow, and often unnecessary compared with prompt updates that can be swapped into code immediately. Source: transcript `[41:22]` to `[44:40]`.
- RAG solves freshness, grounding, and sourcing problems by pairing embeddings, vector databases, retrieval, and a prompt template that constrains answers to the documents. Source: transcript `[45:04]` to `[53:23]`.
- Agentic workflows, memory, tools, MCP, evals, and multi-agent systems turn a single-turn model into a multi-step system with guardrails and debugging points. Source: transcript `[53:30]` to `[1:43:12]`.

## Key Concepts
- **Prompt engineering.** Better task framing, audience, length, style, and output constraints raise model performance. Source: transcript `[22:09]` to `[22:35]`.
- **Jagged frontier.** Some tasks benefit from AI, others get worse when humans rely on it blindly. Source: transcript `[18:59]` to `[19:58]`.
- **Centaurs and cyborgs.** Two work styles: delegate a whole task to the model, or stay in tight back-and-forth augmentation mode. Source: transcript `[20:20]` to `[21:29]`.
- **Chain of thought.** Step-by-step prompting is presented as a popular way to improve reasoning and control. Source: transcript `[24:01]` to `[24:39]`.
- **Prompt templates.** Reusable scaffolds let teams personalize prompts with metadata and ship them in code. Source: transcript `[25:37]` to `[27:37]`.
- **Few-shot prompting.** Adding examples to the prompt is often faster than building a separate dataset and fine tuning. Source: transcript `[28:44]` to `[30:40]`.
- **Prompt chaining.** Separate prompts for extraction, outline, and drafting are easier to debug than one giant prompt. Source: transcript `[32:13]` to `[37:46]`.
- **Fine tuning.** Useful for high-precision domain language, but the lecture warns about data cost, overfitting, and model churn. Source: transcript `[41:31]` to `[43:03]`.
- **RAG.** Retrieval-Augmented Generation embeds docs, stores them in a vector database, retrieves relevant chunks, and inserts them into the prompt. Source: transcript `[45:54]` to `[52:58]`.
- **Agentic workflows.** A workflow is called agentic when prompts, tools, API calls, and resources are chained into a multi-step process. Source: transcript `[54:11]` to `[55:02]`.
- **Memory.** Working memory and archival memory should be split because not every fact needs fast access. Source: transcript `[1:07:30]` to `[1:08:57]`.
- **MCP.** The model context protocol is described as a middle layer that makes tool access more scalable than hand-wiring each API. Source: transcript `[1:12:34]` to `[1:17:12]`.
- **Evals.** End-to-end metrics, component-level checks, objective tests, subjective rubrics, and LLM judges all belong in the stack. Source: transcript `[1:19:11]` to `[1:33:00]`.
- **Multi-agent systems.** Specialized agents are easier to debug and parallelize than one monolith. Source: transcript `[1:34:25]` to `[1:43:12]`.

## Non-Obvious Insights
- The lecture's real thesis is not "use more LLMs." It is "move work into inspectable steps so you can debug the workflow." Source: transcript `[33:14]` to `[37:46]`, `[1:30:27]` to `[1:32:05]`.
- RAG is not only about accuracy. The lecture explicitly values source grounding and developer control. Source: transcript `[46:23]` to `[47:05]`.
- The important unit of optimization is often the prompt or sub-prompt, not the model. Source: transcript `[36:28]` to `[37:03]`.
- Deterministic tasks should stay deterministic. The lecture keeps separating fuzzy model parts from hard-coded business logic. Source: transcript `[58:07]` to `[59:29]`.
- Evals are the bridge from "it feels better" to "we know it is better." The lecture uses both human review and LLM judges. Source: transcript `[1:23:26]` to `[1:25:21]`.
- Multi-agent systems are not magic. They mostly buy reuse, debugging, and parallelization. Source: transcript `[1:42:38]` to `[1:43:12]`.
- The lecture keeps returning to breadth. The field moves too fast to memorize every RAG variant. Source: transcript `[1:49:02]` to `[1:49:36]`.
