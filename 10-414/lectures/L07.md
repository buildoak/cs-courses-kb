# L07 Neural Network Library Abstractions

Lecture 7 is not about a new algorithm. It is about where the framework boundary sits. That is the whole game.

## Summary
- Caffe's `Layer` abstraction couples `forward` and `backward` into one reusable unit, which made early deep learning feel like a stack of composable blocks. Source: transcript `[3:26]` to `[12:19]`.
- TensorFlow 1.0 switches to a declarative graph/session model: declare the computation first, then execute it later with feeds. That buys global optimization and remote execution, but makes debugging awkward. Source: transcript `[14:06]` to `[21:55]`.
- PyTorch and needle flip the model to define-by-run imperative AD. The graph is built while values are computed, so Python control flow and dynamic shapes fit naturally. Source: transcript `[22:16]` to `[32:43]`.
- The course's answer is `nn.Module`: a tensor-in, tensor-out composition layer that collects parameters, wraps loss functions, and lets optimizers and data pipelines plug in cleanly. Source: transcript `[35:14]` to `[55:42]`.
- The key split is responsibility: tensors and autograd handle gradients; modules handle composition. That is the design move the lecture wants you to notice. Source: transcript `[57:21]` to `[59:30]`.

## Key Concepts
- **Layer API.** `forward` writes activations, `backward` writes gradients, and the training loop walks the stack in reverse. Source: transcript `[8:04]` to `[12:19]`.
- **Declarative graph.** TensorFlow 1.0 builds a graph first, then runs it through a session and feed dict. Source: transcript `[16:08]` to `[21:55]`.
- **Define-by-run.** PyTorch and needle compute values as they build the graph, so `if` statements and data-dependent structure are legal. Source: transcript `[24:48]` to `[31:38]`.
- **Static vs dynamic graphs.** Static graphs optimize better; dynamic graphs are easier to debug and easier to express in Python. Source: transcript `[26:39]` to `[32:43]`.
- **`nn.Module`.** A module is a recursive composition unit with parameters attached. Source: transcript `[43:03]` to `[45:08]`.
- **Tensor-in, scalar-out losses.** Loss modules take predictions and labels, return a scalar, and compose by addition. Source: transcript `[44:46]` to `[46:36]`.
- **Train / inference split.** Some modules behave differently when training versus deploying. Source: transcript `[46:41]` to `[47:17]`.
- **Data pipeline modularity.** Data loading and augmentation are also compositional, not a separate side quest. Source: transcript `[53:01]` to `[54:58]`.

## Non-Obvious Insights
- TensorFlow's awkwardness is not accidental. The declarative model exists because global graph knowledge buys optimization, memory reuse, and remote execution.
- PyTorch's win is not better math. It is better ergonomics for researchers who need to print, branch, and mutate on the fly.
- `nn.Module` is really a way to separate "how do we compose?" from "how do we differentiate?".
- The modularity argument is bigger than neural nets. The lecture treats hypothesis class, loss, optimizer, initialization, and data loading as separable knobs.
- Residual blocks are the forcing function. Once a block contains other blocks, recursive module composition becomes the only sane interface.
- The lecture's real thesis is that good abstractions reduce friction in experimentation, which is why deep learning scaled so fast.
