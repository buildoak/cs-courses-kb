# L02 ML Refresher / Softmax Regression

Lecture 2 rebuilds the ML baseline: labels, hypothesis class, loss, optimization, and the matrix form that makes the code fast enough to matter.

## Summary
- The lecture starts with handwritten digit classification to show why hand-written rules break down and supervised learning takes over. Source: transcript `[1:10]` to `[5:10]`.
- Every supervised ML algorithm gets reduced to three pieces: hypothesis class, loss function, and optimization procedure. Source: transcript `[5:39]` to `[7:15]`.
- Softmax regression is the working example: a linear classifier on MNIST, written in batch matrix form so the math and code line up. Source: transcript `[8:45]` to `[22:23]`.
- The 0/1 loss is intuitive but useless for gradient-based training, so the lecture switches to softmax / cross-entropy loss. Source: transcript `[22:34]` to `[35:12]`.
- The final algorithm is plain minibatch gradient descent. Despite the long derivation, the implementation is only a few lines of Python, and it lands at about 8 percent error on MNIST. Source: transcript `[35:30]` to `[1:19:57]`.

## Key Concepts
- **Training data.** A labeled set of examples is the raw material for supervised learning. Source: transcript `[3:37]` to `[4:22]`.
- **Hypothesis class.** The model is a parameterized function `h` that maps inputs to outputs. Source: transcript `[4:28]` to `[7:15]`.
- **Loss function.** The lecture compares the intuitive 0/1 error loss with the trainable cross-entropy loss. Source: transcript `[22:34]` to `[35:12]`.
- **Optimization procedure.** The algorithm finds parameters by minimizing the average loss over the training set. Source: transcript `[35:30]` to `[38:35]`.
- **Batch matrix notation.** `X` is the stacked input matrix, `y` is the label vector, and the batch form is both cleaner and faster. Source: transcript `[17:14]` to `[22:23]`.
- **Softmax.** Exponentiate logits, normalize by the sum, and turn arbitrary scores into a probability distribution. Source: transcript `[27:08]` to `[30:17]`.
- **Cross-entropy.** The loss is `-log P(y | x)`, which expands to `-h_y(x) + log sum_j exp(h_j(x))`. Source: transcript `[31:24]` to `[35:12]`.
- **SGD.** Training proceeds by iterating over minibatches and updating parameters with the gradient. Source: transcript `[1:17:09]` to `[1:19:57]`.
- **Manual gradient hack.** The lecture deliberately uses scalar chain-rule tricks and size matching before moving to autograd. Source: transcript `[56:27]` to `[1:16:21]`.

## Non-Obvious Insights
- The 0/1 loss is the right evaluation metric, but it is the wrong training objective because it is not differentiable. Source: transcript `[25:10]` to `[26:35]`.
- Matrix batching is not just notation. It is the form that maps to efficient CPU and GPU execution. Source: transcript `[17:14]` to `[22:23]`.
- The softmax should live inside the loss, not as a standalone output transform, because that is numerically cleaner. Source: transcript `[30:21]` to `[35:12]`.
- The long gradient derivation is there to expose the shape logic that autograd will later automate. Source: transcript `[56:27]` to `[1:16:21]`.
- A linear model getting about 8 percent error on MNIST is the baseline proof that the machinery already works before neural nets enter. Source: transcript `[1:17:53]` to `[1:18:38]`.
