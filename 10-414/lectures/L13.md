# L13 Hardware Acceleration Implementation

Lecture 13 is the handoff from GPU theory to backend mechanics. The point is not "GPU support" in the abstract. The point is where bytes live, how they move, and which layer owns that decision.

## Summary
- The lecture starts with a GPU notebook setup, then walks through the path from `nd.array(..., device=nd.cuda())` to actual CUDA-backed storage. Source: transcript `[0:00]` to `[5:27]`.
- `NDArray` is a metadata wrapper around raw storage: `shape`, `strides`, `offset`, `device`, and `handle` define the view; the handle is the actual allocation. Source: transcript `[7:08]` to `[16:38]`.
- Scalar arithmetic goes through operator overloading and backend dispatch, while CUDA kernels need explicit launch sizing and a host wrapper. Source: transcript `[17:04]` to `[21:24]`.
- `to_numpy` is a copy-back path, not a view. The lecture shows the host/device split both on the way in and on the way out. Source: transcript `[23:00]` to `[25:00]`.
- Strides, offsets, transpose, slice, and broadcast all become view changes until `compact()` has to materialize contiguous memory. Source: transcript `[26:00]` to `[39:09]`.
- The implementation workflow is intentionally boring: edit, rebuild, run a fresh Python process or test file, and catch stale-import bugs immediately. Source: transcript `[42:17]` to `[47:20]`.

## Key Concepts
- **Backend device.** Dispatch object that routes array allocation and kernels to CPU, NumPy, or CUDA implementations.
- **`NDArray`.** A lightweight object that tracks view metadata rather than owning a Python list of values.
- **`handle`.** The raw pointer or allocation in backend memory.
- **`shape` / `strides` / `offset`.** The trio that defines how a view indexes into the handle.
- **`from_numpy`.** The host-to-device copy path for initial data.
- **`to_numpy`.** The device-to-host copy path for inspection and interop.
- **Scalar kernels.** Elementwise ops that launch a CUDA kernel with grid and block sizing.
- **`compact()`.** The escape hatch that turns a non-contiguous view into contiguous storage.
- **Broadcast via zero stride.** Repetition without copying, represented as metadata.

## Non-Obvious Insights
- The lecture's real boundary is not CPU vs GPU. It is metadata vs storage.
- `reshape`, `slice`, `transpose`, and `broadcast` are cheap because they rewrite interpretation, not bytes.
- `compact()` exists because many downstream kernels want contiguous traversal, even if the view system itself can represent arbitrary layouts.
- The device abstraction is what keeps Needle from becoming a pile of ad hoc copies.
- Re-running a fresh Python process is part of the backend contract because notebook state can hide compilation mistakes.
- The whole setup is a small systems lesson: correctness depends on the metadata contract being precise.
