# L09 NN Library Implementation

Lecture 9 is where the abstractions become concrete code. The course stops talking about "a framework" and starts wiring the pieces together.

## Summary
- Naively updating `w = w - lr * grad` inside the graph leaks memory, because each step keeps the previous computational graph alive. The fix is to detach optimizer state from gradient-tracked tensors. Source: transcript `[4:02]` to `[12:33]`.
- `Parameter` is just a marker subclass of `Tensor`, but it gives `Module.parameters()` a type it can discover recursively through nested modules. Source: transcript `[23:16]` to `[36:20]`.
- Loss functions are modules too. The `tensor in, scalar out` rule lets several objectives compose into one scalar loss that still backpropagates cleanly. Source: transcript `[37:23]` to `[42:08]`.
- The optimizer owns mutable state. SGD is just a step rule, but momentum and similar methods require auxiliary buffers stored alongside the parameter list. Source: transcript `[42:11]` to `[50:39]`.
- Initialization is variance propagation, not folklore. The lecture derives why ReLU halves variance and why the correct sigma depends on fan-in and distribution family. Source: transcript `[50:43]` to `[52:52]`.
- Tensor tuples are the bonus abstraction that makes multi-output ops fit the same autograd engine without special cases. Source: transcript `[53:04]` to `[54:47]`.

## Key Concepts
- **Detached tensors.** Optimizer updates should not keep a gradient history.
- **`Parameter` marker type.** It is a Tensor subclass used for discovery, not new math.
- **Recursive module traversal.** `Module.parameters()` walks nested modules by inspecting `__dict__`.
- **`tensor in, tensor out`.** Modules compose cleanly when their interfaces stay simple.
- **Loss as module.** A loss can be treated like any other layer, except it returns a scalar.
- **Optimizer state.** Momentum lives in the optimizer, not in the model.
- **SGD step rule.** The optimizer loop updates the underlying `data` in place.
- **Fan-in initialization.** Weight scale is derived from variance preservation.
- **Tuple-valued tensors.** Multi-return ops can still participate in autograd.

## Non-Obvious Insights
- The memory bug is not just "Python kept a reference." The deeper issue is that graph-tracked tensors are the wrong container for stateful optimizer updates.
- `Parameter` adds almost no behavior. The important part is the tag that lets the module walker know what counts as trainable state.
- The recursive module design is what makes complex models, including residual-style compositions, feel like a single object.
- Keeping momentum inside the optimizer preserves a clean boundary: model definition stays declarative, optimization policy stays swappable.
- Initialization is framed as a forward-pass variance problem, not as a magical constant picked by convention.
- Tuple outputs are a quiet but important step toward richer operator signatures without fracturing the autograd implementation.
