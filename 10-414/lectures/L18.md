# L18 Transformers Implementation

Lecture 18 is the "the math is short, the tensor shapes are the work" lecture. It turns the Transformer block into NumPy and matches PyTorch to numerical precision.

## Summary
- The first step is a single-head self-attention clone: `softmax(K Q^T / sqrt(d)) V`, plus an output projection. The lecture then checks that NumPy matches PyTorch. Source: transcript `[0:00]` to `[9:54]`.
- PyTorch packs `Wk`, `Wq`, and `Wv` into one projection, then uses a separate output projection. The lecture also points out that `batch_first=True` is a convenience choice, not the most cache-friendly layout. Source: transcript `[10:09]` to `[15:45]`.
- The missing primitive for attention is batch matrix multiplication. Ordinary `matmul` can sometimes be faked with reshape tricks, but true batched attention needs a `bmm`-style operation. Source: transcript `[16:01]` to `[30:48]`.
- Multi-head attention is just several smaller self-attention blocks over column partitions of the embedding space, then a reshape back to the full embedding size. Source: transcript `[31:03]` to `[44:42]`.
- The Transformer block is self-attention, residual, layer norm, feed-forward, residual, layer norm. The whole implementation is compact once the tensor plumbing is right. Source: transcript `[45:02]` to `[1:00:30]`.

## Key Concepts
- **Stable softmax.** Subtracting the row max before exponentiation keeps the attention weights numerically sane. Source: transcript `[4:37]` to `[5:20]`.
- **Packed projections.** A single `WKQV` matrix is split into `K`, `Q`, and `V` after the multiply. Source: transcript `[6:00]` to `[8:18]`.
- **Attention mask.** A triangular `-inf` mask enforces causality after the softmax. Source: transcript `[11:07]` to `[12:27]`.
- **Batch-first vs time-major.** The lecture prefers batch-first for the Transformer implementation because the batch is the leading axis for the batched matmuls. Source: transcript `[16:10]` to `[19:02]`.
- **`bmm` primitive.** Batched attention needs an operator that multiplies each batch slice independently. Source: transcript `[20:13]` to `[30:48]`.
- **Head splitting.** Multi-head attention reshapes `B x T x D` into `B x H x T x (D/H)` before the attention step. Source: transcript `[37:40]` to `[41:48]`.
- **Residual + norm.** The Transformer block alternates self-attention, residual addition, layer norm, and feed-forward MLPs. Source: transcript `[45:02]` to `[49:38]`.
- **Encoder layer.** The lecture uses `TransformerEncoderLayer` as the PyTorch reference because the original encoder-decoder setup is overkill for this example. Source: transcript `[49:43]` to `[53:25]`.

## Non-Obvious Insights
- The packed projection is not just prettier code. It keeps the matrix multiply large and contiguous, which is the real performance win. Source: transcript `[6:00]` to `[7:24]`.
- `matmul` and `bmm` are not the same thing. One can often be simulated by reshape gymnastics; the other is an actual batched operator over leading dimensions. Source: transcript `[23:44]` to `[26:38]`.
- `batch_first=True` is there for API compatibility with RNNs. The lecture explicitly says it is not the computationally natural layout for attention. Source: transcript `[18:01]` to `[19:02]`.
- PyTorch's returned attention shape is not the raw per-head tensor the lecture computes. It averages across heads in the module API. Source: transcript `[43:36]` to `[44:42]`.
- The core Transformer math is short. Most of the difficulty is shape discipline, not algorithm design. Source: transcript `[59:00]` to `[1:00:30]`.
