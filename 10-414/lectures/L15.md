# L15 Sequence Modeling + RNNs

Lecture 15 is where sequence data stops pretending to be IID. Order matters, memory matters, and the whole point of the model is to carry context forward.

## Summary
- The lecture begins by separating sequential data from IID examples and uses part-of-speech tagging, speech-to-text, and autoregressive prediction as the motivating cases. Source: transcript `[0:00]` to `[11:14]`.
- A vanilla RNN maintains hidden state across time, and training is just backpropagation through time over the unrolled computation graph. Source: transcript `[12:00]` to `[24:33]`.
- Plain RNNs still hit exploding and vanishing gradients, especially once recurrence and saturating activations interact. Source: transcript `[27:29]` to `[38:37]`.
- LSTMs split hidden state into hidden and cell components and use gate vectors to keep memory alive longer. Source: transcript `[39:50]` to `[54:54]`.
- The lecture is blunt about the history: the 1997 Hochreiter-Schmidhuber LSTM paper matters, and Karpathy's 2015 blog post helped popularize practical RNN/LSTM use. Source: transcript `[55:06]` to `[57:27]`.
- Sequence-to-sequence and bidirectional RNNs extend the same module idea to translation and whole-sequence context. Source: transcript `[1:02:33]` to `[1:10:12]`.

## Key Concepts
- **IID vs sequence.** Sequential data has order-dependent structure that random shuffling destroys.
- **Autoregressive prediction.** Predict the next token from the tokens seen so far.
- **Hidden state.** The recurrent summary vector that carries context across time.
- **BPTT.** Backpropagation through time, the standard way to train an unrolled RNN.
- **Exploding gradient.** Gradients can blow up as they pass through many recurrent steps.
- **Vanishing gradient.** Gradients can decay toward zero before they reach early time steps.
- **LSTM cell state.** The memory path that survives longer than a vanilla hidden state.
- **Gate vectors.** Sigmoid-controlled terms that decide what to keep, write, or expose.
- **Sequence-to-sequence.** Encoder-decoder composition for mapping one sequence to another.
- **Bidirectional RNN.** Forward and backward recurrence combined when future context is allowed.

## Non-Obvious Insights
- RNN training is only simple because autodiff already knows how to backprop through the unrolled graph.
- The hidden state is a summary vector, not a magic memory box.
- The cell state is the real long-term memory path in an LSTM.
- The lecture is skeptical of gate folklore. The gate names matter less than the additive form of the update.
- Sequence-to-sequence works because an RNN can be used as an encoder module and a decoder module in one graph.
- Bidirectional RNNs are appropriate only when the task can use future context.
- Automatic differentiation is the only sane way to handle the gradient bookkeeping once the graph gets this tangled.
