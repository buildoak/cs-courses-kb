# L14 Convolutional Network Implementation

Lecture 14 turns convolution into a memory-layout problem. The math is already known. The work here is making the data sit in the right shape so the math becomes cheap.

## Summary
- Convolution is first framed as a storage-order question: images are stored in NHWC form, and filters are stored as `K x K x C_in x C_out` tensors. Source: transcript `[0:00]` to `[9:11]`.
- The lecture writes a PyTorch reference implementation first, then a naive Python convolution, then a 1x1 matrix-multiply version, and finally a tiled `as_strided` version. Source: transcript `[10:06]` to `[1:19:09]`.
- A 1x1 convolution collapses to a plain matrix multiply over channels, which is the bridge to the larger convolution rewrite. Source: transcript `[26:00]` to `[30:40]`.
- `as_strided` turns an image into overlapping windows without copies; reshape then flattens those windows into a matrix for GEMM. Source: transcript `[38:11]` to `[1:07:55]`.
- The practical win is not just elegance; it is speed. The optimized `as_strided` path is roughly 2x faster than the earlier tiled version in the lecture's benchmark. Source: transcript `[1:09:12]` to `[1:16:43]`.

## Key Concepts
- **Storage order.** NHWC vs NCHW is not a cosmetic choice; it changes how easy it is to turn convolution into matrix multiplication.
- **Filter layout.** The kernel tensor is indexed by spatial location, input channel, and output channel.
- **Reference implementation.** PyTorch is used as the shape-and-value baseline before Needle code is trusted.
- **1x1 convolution.** A convolution with kernel size 1 is just a matrix multiply over channels.
- **Tiling.** The lecture exposes all local windows by reinterpreting the input as a tiled matrix.
- **`as_strided`.** A view constructor that reuses memory with new strides.
- **Contiguous copy.** `reshape` materializes a contiguous matrix when the logical view has overlapping storage.
- **Batch matmul.** The final formulation compresses convolution into one big matrix multiplication after window extraction.
- **No padding.** The demo strips padding out so the stride logic stays visible.

## Non-Obvious Insights
- The real trick is not convolution math. It is memory layout.
- `as_strided` makes window extraction free because it only changes metadata.
- The expensive step is the reshape that compacts overlapping windows into a true matrix.
- The lecture's speedup comes from turning many small multiplies into one larger matmul.
- `NHWC` is chosen because the channel dimension sits naturally at the end of the matrix product.
- A "fast convolution" usually means "a good memory story," not a clever loop nest.
- The no-padding constraint is deliberate; it keeps the stride logic legible.
