# L19 Training Large Models

Lecture 19 is the "memory first, scale second" lecture. It starts with the GPU memory wall, then shows the standard escapes: checkpointing, model parallelism, pipeline parallelism, and data parallelism.

## Summary
- Training large models is mostly a memory problem before it is a compute problem: weights, optimizer state, activations, and input data all consume GPU RAM. Source: transcript `[2:54]` to `[6:18]`.
- Inference can recycle two buffers in a ping-pong pattern, but training cannot because backward pass dependencies need the forward activations. Source: transcript `[7:00]` to `[13:16]`.
- Activation checkpointing, also called rematerialization, drops intermediate activations and recomputes them during backward pass, trading about 25% extra compute for sublinear memory. Source: transcript `[14:59]` to `[25:54]`.
- Model parallelism splits the computation graph across workers, and pipeline micro-batches keep those workers busy. Source: transcript `[26:03]` to `[32:10]`.
- Data parallelism replicates the model across workers and synchronizes gradients with allreduce or a parameter server, ideally overlapping communication with backprop. Source: transcript `[32:10]` to `[46:31]`.

## Key Concepts
- **Memory sources.** The lecture separates memory into model weights, optimizer state, intermediate activations, and data. That breakdown is the real starting point for scaling. Source: transcript `[4:51]` to `[6:18]`.
- **Inference reuse.** For a deep feed-forward net, inference can alternate between two scratch buffers because each layer only needs the previous output. Source: transcript `[7:00]` to `[9:56]`.
- **Training dependency.** Reverse-mode autodiff keeps forward values alive, so naive training needs activation memory that grows with depth. Source: transcript `[11:12]` to `[13:16]`.
- **Activation checkpointing.** Save only selected checkpoints, recompute the missing values during backward, then free them again. Source: transcript `[14:59]` to `[25:54]`.
- **Sublinear memory.** If you checkpoint every `k` layers and pick `k = sqrt(n)`, total activation memory becomes `O(sqrt(n))` instead of `O(n)`. Source: transcript `[21:45]` to `[22:48]`.
- **Model parallelism.** Partition the graph by worker and pass tensors across boundaries with send/receive pairs. Source: transcript `[27:24]` to `[29:21]`.
- **Pipeline parallelism.** Break work into micro-batches so later workers process one batch while earlier workers start the next. Source: transcript `[29:27]` to `[31:13]`.
- **Allreduce.** Data parallel training uses collective gradient summation across replicas. The lecture names NCCL, Horovod, and BytePS as concrete implementations. Source: transcript `[33:55]` to `[36:06]`.
- **Parameter server.** The server can absorb gradients, update weights centrally, and tolerate stragglers or worker restarts more gracefully. Source: transcript `[38:05]` to `[41:08]`.
- **Async overlap.** The lecture pushes asynchronous allreduce so communication starts as soon as gradients are ready instead of waiting for the full backward pass. Source: transcript `[41:16]` to `[44:46]`.

## Non-Obvious Insights
- Training scale is not just "bigger model, bigger GPU." The bottleneck is usually activation storage, not raw matrix multiply throughput. Source: transcript `[3:01]` to `[6:18]` and `[11:12]` to `[13:16]`.
- Checkpointing is a compute-memory exchange, not a free lunch. The lecture is explicit that the gain comes from recomputation, not from eliminating work. Source: transcript `[20:34]` to `[25:54]`.
- Pipeline parallelism only pays if you have enough micro-batches to fill the bubbles. Without that, partitioning alone leaves workers idle. Source: transcript `[29:27]` to `[32:10]`.
- Data parallelism is mostly a synchronization problem once the local gradients exist. The hard part becomes hiding communication behind computation. Source: transcript `[41:16]` to `[44:46]`.
- The lecture frames distributed training as system design, not model design. The model stays recognizable; the graph partition, checkpoint policy, and collective schedule are what change. Source: transcript `[26:03]` to `[46:31]`.
