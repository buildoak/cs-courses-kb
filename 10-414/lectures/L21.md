# L21 Generative Models Implementation

Lecture 21 is the working code version of the GAN story. It builds the generator, discriminator, and training loop in Needle, then shows how to wrap the discriminator update inside a modular loss object.

## Summary
- The demo starts with a 2D Gaussian target distribution and trains a model to reproduce that distribution from latent noise. Source: transcript `[2:27]` to `[7:50]`.
- The generator is first a simple linear map from random noise to fake samples; the discriminator is a small MLP trained with softmax loss. Source: transcript `[4:04]` to `[9:56]`.
- Generator updates feed fake samples through the discriminator and optimize them as if they were real; discriminator updates classify fake as zero and real as one. Source: transcript `[10:11]` to `[17:50]`.
- The full training loop alternates both updates over epochs and batches, then shows that the learned generator matches the target distribution shape. Source: transcript `[18:07]` to `[25:57]`.
- The second half turns the GAN into a modular loss object, so the outer training loop looks like ordinary supervised training. Source: transcript `[26:27]` to `[35:22]`.

## Key Concepts
- **Toy target distribution.** The real data is a 2D Gaussian with covariance `A^T A` and a mean vector, then visualized with a scatter plot. Source: transcript `[2:27]` to `[3:56]`.
- **Generator.** The initial model `G` is a linear network that maps latent noise into the data space. Source: transcript `[4:21]` to `[6:16]`.
- **Discriminator.** A 3-layer neural network separates real samples from fake ones. Source: transcript `[8:02]` to `[9:14]`.
- **Binary loss via softmax.** The lecture uses softmax loss as a two-class classifier for the discriminator. Source: transcript `[9:21]` to `[10:00]`.
- **Generator update.** Fake samples are labeled as real so the generator learns to fool the discriminator. Source: transcript `[10:51]` to `[14:18]`.
- **Discriminator update.** Fake samples are detached, then fake labels are zero and real labels are one. Source: transcript `[14:42]` to `[17:50]`.
- **Alternating training loop.** The outer loop cycles over epochs, batches, generator updates, and discriminator updates. Source: transcript `[18:07]` to `[25:44]`.
- **Modular GAN loss.** The loss object hides discriminator updating inside `forward`, which makes the outer code look like a normal model-plus-loss pipeline. Source: transcript `[26:27]` to `[31:38]`.
- **Distribution fit, not parameter identity.** The learned generator matches the target covariance and mean, but the internal factorization need not equal the original matrix `A`. Source: transcript `[24:28]` to `[25:35]`.

## Non-Obvious Insights
- The implementation lesson is mostly about gradient routing. `detach` is the key detail that keeps discriminator updates from leaking back into the generator. Source: transcript `[15:45]` to `[16:05]`.
- GAN training does not need the original parameterization to be recovered. Matching the distribution is enough, which is why the learned `G` can differ from the source matrix while still succeeding. Source: transcript `[24:28]` to `[25:35]`.
- Turning GAN into a loss object is a deliberate abstraction move. It hides the adversarial update inside the module interface so higher-level composition stays clean. Source: transcript `[26:27]` to `[35:22]`.
- The modular version is the practical payoff of lecture 20. Once the loss behaves like a standard component, GANs compose with the same machinery as supervised models. Source: transcript `[31:42]` to `[35:22]`.
- The code is a framework argument in disguise: Needle is good when it can express both standard supervised pipelines and adversarial ones without special cases. Source: transcript `[34:18]` to `[35:22]`.
