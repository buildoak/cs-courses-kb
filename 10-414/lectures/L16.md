# L16 Sequence Modeling Implementation

Lecture 16 is the "make the RNN real, then stop pretending 10,000-step backprop is free" lecture.

## Summary
- The lecture starts by cloning a PyTorch LSTM cell in NumPy. The point is not novelty; it is to show that the gate math is only a few matrix multiplies plus nonlinearities. Source: transcript `[0:00]` to `[13:42]`.
- A full LSTM is just that same cell unrolled across time. PyTorch returns the whole hidden-state sequence plus the last cell state, which is exactly the state you need for continued training. Source: transcript `[14:14]` to `[22:17]`.
- Batched LSTMs flip the usual layout. Time comes first, batch second, because the time slice needs to stay contiguous in memory for efficient matrix multiplies. Source: transcript `[22:51]` to `[35:22]`.
- Training is ordinary autograd until the sequence gets long. Then the graph blows up, so the lecture switches to truncated backpropagation through time and detached hidden-state carryover. Source: transcript `[36:22]` to `[54:11]`.

## Key Concepts
- **Packed gates.** One affine output is split into `i`, `f`, `g`, and `o`, then each chunk gets its own nonlinearity. Source: transcript `[6:24]` to `[9:20]`.
- **Cell update rule.** `c_out = f * c + i * g`, then `h_out = o * tanh(c_out)`. Source: transcript `[8:18]` to `[9:20]`.
- **Sequence unrolling.** The full LSTM is the cell applied repeatedly over `x_t` in order. Source: transcript `[17:10]` to `[22:17]`.
- **Time-major batching.** For LSTMs, the time axis moves to the front so a timestep slice stays contiguous. Source: transcript `[24:52]` to `[29:40]`.
- **Deep stacking.** Multi-layer RNNs are usually run layer-by-layer over the whole sequence, not timestep-by-timestep across layers. Source: transcript `[39:57]` to `[43:26]`.
- **Truncated BPTT.** Long sequences are chopped into blocks so the autograd graph does not span the whole history. Source: transcript `[44:55]` to `[47:03]`.
- **Hidden-state repackaging.** `h` and `c` are copied forward as detached values, not as live graph nodes. Source: transcript `[47:40]` to `[54:11]`.

## Non-Obvious Insights
- PyTorch's LSTM docs look more complicated than the implementation because the library packs the four gate matrices into two big blocks. Source: transcript `[2:35]` to `[5:36]`.
- The batch-order choice is about cache behavior, not style. The lecture changes the layout because a time slice must be contiguous before matrix multiplication. Source: transcript `[26:19]` to `[29:40]`.
- Returning the last hidden and cell states is not cosmetic. That is the clean boundary that makes truncated BPTT work. Source: transcript `[52:13]` to `[52:24]`.
- Repackaging is a state handoff, not a gradient handoff. The next block gets the value, but not the compute graph. Source: transcript `[49:20]` to `[50:12]`.
- The "hard" part of long-sequence training is graph management. The recurrent math is not the bottleneck. Source: transcript `[44:04]` to `[45:00]`.
