# 10-414 Gems

## Surprising Connections
- `detach` in optimizer updates, hidden-state repackaging in truncated BPTT, and activation checkpointing are the same idea at different scales: preserve values, cut history. Sources: `/Users/otonashi/thinking/building/cs-courses-kb/10-414/lectures/L09.md`, `/Users/otonashi/thinking/building/cs-courses-kb/10-414/lectures/L16.md`, `/Users/otonashi/thinking/building/cs-courses-kb/10-414/lectures/L19.md`.
- `im2col` for convolutions and `bmm` for attention are siblings. Both rewrite local interactions as batched dense linear algebra so kernels can stay large and contiguous. Sources: `/Users/otonashi/thinking/building/cs-courses-kb/10-414/lectures/L10.md`, `/Users/otonashi/thinking/building/cs-courses-kb/10-414/lectures/L14.md`, `/Users/otonashi/thinking/building/cs-courses-kb/10-414/lectures/L18.md`.
- BatchNorm’s train/test split and deployment export share the same pain: the thing you use while training is not identical to the thing you run in production. Sources: `/Users/otonashi/thinking/building/cs-courses-kb/10-414/lectures/L08.md`, `/Users/otonashi/thinking/building/cs-courses-kb/10-414/lectures/L23.md`.
- `nn.Module` recursion, residual blocks, Transformer blocks, and modular GAN losses all solve the same problem: where to insert composition so the rest of the system stays swappable. Sources: `/Users/otonashi/thinking/building/cs-courses-kb/10-414/lectures/L07.md`, `/Users/otonashi/thinking/building/cs-courses-kb/10-414/lectures/L09.md`, `/Users/otonashi/thinking/building/cs-courses-kb/10-414/lectures/L18.md`, `/Users/otonashi/thinking/building/cs-courses-kb/10-414/lectures/L20.md`, `/Users/otonashi/thinking/building/cs-courses-kb/10-414/lectures/L21.md`.
- RNN recurrence, CNN receptive fields, and Transformer attention are three answers to the same question: how far back can a token or pixel see, and at what cost? Sources: `/Users/otonashi/thinking/building/cs-courses-kb/10-414/lectures/L10.md`, `/Users/otonashi/thinking/building/cs-courses-kb/10-414/lectures/L15.md`, `/Users/otonashi/thinking/building/cs-courses-kb/10-414/lectures/L16.md`, `/Users/otonashi/thinking/building/cs-courses-kb/10-414/lectures/L17.md`, `/Users/otonashi/thinking/building/cs-courses-kb/10-414/lectures/L18.md`.
- Large-model training and hardware acceleration are the same memory story at different scales. CPU caches, GPU shared memory, activation checkpointing, and pipeline micro-batches all trade compute for locality. Sources: `/Users/otonashi/thinking/building/cs-courses-kb/10-414/lectures/L11.md`, `/Users/otonashi/thinking/building/cs-courses-kb/10-414/lectures/L12.md`, `/Users/otonashi/thinking/building/cs-courses-kb/10-414/lectures/L19.md`.
- Pretrained-model customization and GAN modularization both push control as close as possible to the cheapest interface that still changes behavior. Sources: `/Users/otonashi/thinking/building/cs-courses-kb/10-414/lectures/L20.md`, `/Users/otonashi/thinking/building/cs-courses-kb/10-414/lectures/L21.md`, `/Users/otonashi/thinking/building/cs-courses-kb/10-414/lectures/L22.md`.

## Under-Appreciated Ideas
- Numerical differentiation is mostly a test harness. The real value of autodiff is that it makes graph-shaped computation first-class and higher-order gradients possible. Sources: `/Users/otonashi/thinking/building/cs-courses-kb/10-414/lectures/L04.md`, `/Users/otonashi/thinking/building/cs-courses-kb/10-414/lectures/L05.md`.
- The hard part of deep learning is often not the algorithm, but the memory contract. Missing buffers, cached activations, stale graphs, and batch-dependent statistics cause more trouble than the math does. Sources: `/Users/otonashi/thinking/building/cs-courses-kb/10-414/lectures/L05.md`, `/Users/otonashi/thinking/building/cs-courses-kb/10-414/lectures/L08.md`, `/Users/otonashi/thinking/building/cs-courses-kb/10-414/lectures/L09.md`, `/Users/otonashi/thinking/building/cs-courses-kb/10-414/lectures/L16.md`, `/Users/otonashi/thinking/building/cs-courses-kb/10-414/lectures/L19.md`.
- `compact()` is as important as `reshape`. The course keeps showing that metadata-only views are cheap until a kernel needs contiguous memory. Sources: `/Users/otonashi/thinking/building/cs-courses-kb/10-414/lectures/L13.md`, `/Users/otonashi/thinking/building/cs-courses-kb/10-414/lectures/L14.md`.
- Transformer implementation is operator engineering, not algorithm invention. The missing primitive is often `bmm`; the rest is shape bookkeeping and stable softmax. Sources: `/Users/otonashi/thinking/building/cs-courses-kb/10-414/lectures/L17.md`, `/Users/otonashi/thinking/building/cs-courses-kb/10-414/lectures/L18.md`.
- The course is unusually honest about uncertainty. BatchNorm is still partly unexplained, and the lecture says so instead of forcing a tidy story. Sources: `/Users/otonashi/thinking/building/cs-courses-kb/10-414/lectures/L08.md`.
- Deployment is compilation plus search, not just export. Operator fusion, lowering, and auto-tuning are where runtime wins actually happen. Sources: `/Users/otonashi/thinking/building/cs-courses-kb/10-414/lectures/L23.md`.
- Customization is a placement problem. The surface can be weights, embeddings, latent conditioning, prompts, or logits, and the right answer depends on the cheapest point of leverage. Sources: `/Users/otonashi/thinking/building/cs-courses-kb/10-414/lectures/L22.md`.
- Sequence-state management is a general pattern, not an RNN quirk. Detaching hidden state, truncating BPTT, and carrying optimizer state all use the same "keep the value, drop the graph" boundary. Sources: `/Users/otonashi/thinking/building/cs-courses-kb/10-414/lectures/L09.md`, `/Users/otonashi/thinking/building/cs-courses-kb/10-414/lectures/L16.md`.
