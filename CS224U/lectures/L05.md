# CS224U L05: Contextual Representations, Part 2

## Summary
- The lecture opens the transformer by adding word embeddings and position embeddings to get position-sensitive inputs.
- Scaled dot-product attention is then used to compare each token to the others, normalize the scores, and build a weighted context vector.
- Multi-head attention adds separate query, key, and value projections so the model can look at the sequence in parallel ways.
- The original transformer is still an encoder-decoder machine: the decoder just uses masked attention so it cannot look into the future.
- A BERT Base inspection makes the architecture concrete: 30k-ish vocabulary, 512 positional embeddings, 768-wide states, and a 3072-wide feed-forward expansion.

## Key Concepts
- **Position-sensitive input:** word vectors plus positional vectors become the first contextual representations.
- **Scaled dot-product attention:** dot products are rescaled by `sqrt(dk)` before softmax.
- **Attention weights:** the softmaxed scores decide how much each prior state contributes to the current state.
- **Query / key / value projections:** learned matrices increase representational power inside each attention head.
- **Multi-head attention:** several attention computations run in parallel and are reassembled afterward.
- **Encoder-decoder rhythm:** attention, residual-style mixing, normalization, feed-forward, repeat.
- **Masked attention:** decoding cannot attend to future states.
- **Model inspection:** Hugging Face model graphs expose the repeated block structure instead of hiding it behind the paper diagram.

## Non-Obvious Insights
- Attention is not a decorative add-on. It is the place where sequence parts actually interact.
- The matrix notation in the papers is just a compressed version of the same piecewise calculation shown in the lecture.
- Multi-head attention is not conceptual noise. It gives the model diverse interaction channels without changing the basic block rhythm.
- The feed-forward layer matters because the model needs a capacity bump after attention has mixed information.
- The BERT diagram becomes readable once you notice the repeated block and the fixed dimensionality constraint that makes stacking possible.

## Connections
- This lecture turns the historical arc from L04 into a concrete transformer block.

- It also connects to later BERT-based fine tuning in the course, where these contextual states become classifier inputs.
