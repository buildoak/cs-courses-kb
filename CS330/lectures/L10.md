# CS330 L10: Large-Scale Meta-Optimization

## Summary
- Meta-learning is only useful at scale if the inner-loop computation graph stays tractable.
- The hard limit is often memory, not raw compute: unrolled inner loops and second-order gradients explode graph size.
- Meta-parameters are broader than just initialization. The lecture treats learning rates, optimizers, loss functions, and even datasets as things you can meta-learn.
- Truncated backprop, evolution strategies, implicit differentiation, and forward-mode differentiation are the main escape hatches.
- The real question is not "can we meta-learn?" but "can we differentiate through the meta-learning procedure without blowing up the machine?"

## Key Concepts
- **Direct backpropagation.** The default recipe for black-box, optimization-based, and non-parametric meta-learning.
- **Computation graph size.** Memory scales with the full unrolled inner loop, not just the base model size.
- **Meta-parameters.** Initial weights, learning rates, optimizers, loss functions, datasets, and other parts of the learning pipeline.
- **Truncated backpropagation.** Detach older steps to trade correctness for bounded memory.
- **Gradient checkpointing.** Recompute instead of storing everything; saves memory at the cost of time.
- **Evolution strategies.** Gradient-free search over parameter populations.
- **Implicit differentiation.** Differentiate through the optimum condition instead of the whole optimization path.
- **Forward-mode differentiation.** Push derivatives forward instead of backpropagating them backward.
- **Second-order optimization.** A major source of graph blow-up when the inner loop itself is differentiated.

## Non-Obvious Insights
- The lecture's core scaling failure is structural: all the usual meta-learning methods still build one big graph and backprop through it.
- Learning the loss or the dataset is not a side quest. It is just meta-learning another node in the graph.
- Truncated backprop is not a free approximation. It buys memory by dropping long-range dependencies and introducing bias.
- Evolution strategies look crude, but they are attractive because the graph disappears from memory and the population can run in parallel.
- The best large-scale trick depends on what is expensive: graph depth, parameter dimension, or differentiability.
- Implicit differentiation is elegant, but it only works cleanly when the optimization really reaches an optimum.
