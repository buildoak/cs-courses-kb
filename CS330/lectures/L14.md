# CS330 L14: Domain Generalization

## Summary
- Domain generalization asks for zero-shot performance on an unseen target domain, with no target data available at training time.
- The lecture contrasts this with domain adaptation and meta-learning: only `p(x)` changes across domains, but test-time adaptation is not allowed.
- Regularizer-based methods try to learn domain-invariant features by suppressing spurious correlations.
- CORAL aligns feature covariances across domains, but the regularizer can be too harsh if it removes useful signal.
- Augmentation-based methods use Mixup and LISA to manufacture harder training examples and push the model toward invariant structure.

## Key Concepts
- **Unseen target domain.** Generalize without touching the target data.
- **Spurious correlation.** Background, color, or style leaks label information.
- **Domain invariance.** Learn features that stay stable across domains.
- **Regularizer.** Add an explicit penalty to the loss.
- **CORAL.** Correlation alignment via covariance matching.
- **Mixup.** Interpolate two examples and interpolate their labels.
- **LISA.** A Mixup variant that selectively mixes across labels or domains.
- **Intra-label LISA.** Same label, different domains.
- **Intra-domain LISA.** Same domain, different labels.
- **Manifold Mixup.** Mix in feature space rather than raw input space.
- **Colored MNIST.** The toy example for spurious color-label coupling.

## Non-Obvious Insights
- Regularizers can be more important than model architecture. A bad regularizer can hurt more than a smaller model.
- CORAL can help when the domains have a compatible shape, but it can fail when the distributions are too misaligned.
- Vanilla Mixup is not enough on its own. It augments data, but does not always enforce invariance.
- LISA works by mixing in a way that directly attacks spurious cues, not just by generating more samples.
- The best augmentation strategy can depend on the dataset statistics, especially when domain labels are available.

## Connections
- `Lecture 13` is the transductive version of the same shift problem.
- `Lecture 15` studies sequential learning, where shift arrives over time instead of across fixed domains.
- `IRM`, `IB-IRM`, and `REx` are the other regularizer-heavy baselines the lecture compares against.
- `Manifold Mixup` is the feature-level variant that makes Mixup easier to use for text.
- The benchmarks in the lecture, like OfficeHome, DomainNet, iWildCAM, Camelyon, FMoW, and Amazon, are all examples of domain shift under different modalities.
