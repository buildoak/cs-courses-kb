# L01 What is Multi-Task Learning?

The opening lecture frames the course around shared structure: one model, many tasks, and the question of when prior experience actually pays off.

## Summary
- The course is about learning across tasks, not just fitting one task at a time. Chelsea Finn sets the arc as theory, PyTorch implementation, and a glimpse of the research process behind the methods. Source: `[0:04]` to `[5:28]`.
- Deep learning wins like ImageNet and GNMT explain why the course uses large-scale examples, but the real motivation is the long tail: medical imaging, robotics, personalized education, and rare languages where data is scarce or expensive. Source: `[30:51]` to `[32:47]`.

- Modern examples include Flamingo, a student-feedback system for Code in Place, a YouTube recommendation system, Gato, and robot manipulation. The common theme is broad transfer, not narrow specialization. Source: `[41:42]` to `[50:22]`.

## Key Concepts
- **Multi-task learning.** A model is trained to solve several tasks, often by sharing representations across them. Source: `[4:05]` to `[4:17]`.
- **Few-shot learning.** The model or human uses prior experience to solve a new task from very little data. Source: `[36:03]` to `[37:09]`.
- **Shared structure.** The useful thing is not identical labels or losses; it is the latent structure that lets tasks benefit from each other. Source: `[1:06:09]` to `[1:07:23]`.
- **Transfer learning.** Learn on one task, then use that knowledge to adapt to another. Source: `[1:09:09]` to `[1:10:55]`.
- **Meta-learning.** Optimize for the ability to learn new tasks quickly from a family of source tasks. Source: `[1:09:33]` to `[1:10:55]`.
- **Task distribution.** Meta-learning only makes sense if train tasks and test tasks are drawn from the same broader task distribution. Source: `[1:15:32]` to `[1:16:09]`.
- **Inductive bias.** The lecture treats learned structure as a bias that can be useful when it matches the new task family. Source: `[1:06:09]` to `[1:09:08]`.
- **Generalist systems.** A single model can span dialogue, Atari, robotics, and robot arms if the shared structure is real enough. Source: `[49:17]` to `[50:22]`.

## Non-Obvious Insights
- The real question is not "can one model do many tasks?" It is "is there enough shared structure to make sharing worth it?" Source: `[48:48]` to `[50:22]`, `[1:06:09]` to `[1:10:55]`.
- Task similarity is still an open problem. The lecture says there is no closed-form way to know in advance whether multitask learning will help. Source: `[56:33]` to `[57:23]`.
- A model that cannot generalize to a new task is often just carving its capacity into separate compartments instead of learning a unified representation. Source: `[1:06:32]` to `[1:07:20]`.
- Multi-task learning and single-task learning are not opposites. Multi-task can collapse into a summed-loss single-task formulation, but transfer learning stays the more interesting case. Source: `[1:10:01]` to `[1:11:10]`.
- The lecture is explicit that the process matters as much as the result. Understanding how the algorithms were built is part of the point. Source: `[3:30]` to `[3:56]`.
