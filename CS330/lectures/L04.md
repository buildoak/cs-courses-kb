# L04: Black-Box Meta-Learning

Black-box meta-learning treats the learning process itself as the object being learned: give the model a task dataset, get back a task-aware predictor. Source: `[13:24]` to `[18:20]`.

## Summary
- The lecture re-casts meta-learning as learning a function from a task dataset to predictions on new examples, with support/query splits and a task distribution assumption. Source: `[0:51]` to `[1:55]`, `[9:09]` to `[13:24]`.
- Black-box methods implement that function with recurrent networks, deep sets, attention, or transformers, so the model can read a set and emit task-specific behavior. Source: `[18:03]` to `[18:23]`, `[1:03:57]` to `[1:04:47]`.
- Omniglot is the canonical few-shot benchmark here, and the homework target is to make the model solve few-shot classification on it. Source: `[1:02:09]` to `[1:07:23]`.
- GPT-3 is the modern black-box analogy: few-shot behavior emerges from a large language model trained on text, not from explicit episodic supervision. Source: `[1:09:26]` to `[1:17:30]`.
- The upside is expressiveness and easy composition with supervised learning or RL; the downside is optimization difficulty and weaker data efficiency. Source: `[1:07:38]` to `[1:09:06]`.

## Key Concepts
- **Task distribution.** Meta-learning assumes tasks come from a shared distribution, just as examples in ordinary ML do. Source: `[0:51]` to `[1:55]`.
- **Support set / query set.** The task-specific training data is the support set; the held-out examples are the query set used to judge adaptation. Source: `[9:09]` to `[9:29]`.
- **k-shot, n-way.** The lecture walks through one-shot, five-way classification and the classification/regression distinction in the notation. Source: `[9:36]` to `[12:00]`, `[20:18]` to `[20:30]`.
- **Function f(theta).** The meta-learner is a function that reads a dataset and a new input, then predicts the new label. Source: `[13:01]` to `[18:23]`.
- **Meta-parameters.** Theta stores the learned parameters; in some variants the model also compresses the task into a low-dimensional context vector h_i. Source: `[18:26]` to `[23:07]`, `[58:22]` to `[59:08]`.
- **Set encoders.** RNNs, LSTMs, deep sets, attention, convolutions, and transformers are all ways to make the support set legible to the model. Source: `[1:02:09]` to `[1:04:47]`.
- **Emergent in-context learning.** GPT-3 shows that large text models can behave like black-box meta-learners without being trained on explicit task episodes. Source: `[1:10:03]` to `[1:17:30]`.

## Non-Obvious Insights
- The model is not just classifying. It is learning a learning procedure, which is a harder problem and a richer target. Source: `[13:24]` to `[18:23]`.
- Order matters unless the architecture removes it. Deep sets are the lecture's reminder that support sets are sets, not sequences. Source: `[1:03:57]` to `[1:04:12]`.
- GPT-3's few-shot behavior is a scale effect, not an explicit meta-learning objective. The context window becomes the support set by accident and by design at once. Source: `[1:12:33]` to `[1:17:30]`.
- Black-box learners can be dropped into RL because the inner computation graph can stand in for almost any learning process. Source: `[1:07:57]` to `[1:08:13]`.
- The low-dimensional task vector h_i is basically a learned task descriptor. It is a compressed way to carry context forward when outputting full parameters is too expensive. Source: `[58:22]` to `[1:01:02]`.

## Connections
- **To L05.** Optimization-based meta-learning replaces the black-box inner learner with gradient descent, keeping the same outer meta-objective but adding structure. Source: `[1:09:07]` to `[1:09:24]`.
- **To L06.** Non-parametric methods keep the support set explicit and turn the problem into metric learning and nearest-neighbor classification. Source: `[1:07:09]` to `[1:09:06]`.
- **To Homework 1.** The Omniglot few-shot setup is the implementation target for the black-box family. Source: `[1:02:09]` to `[1:07:23]`.
- **To modern prompting.** In-context learning is black-box meta-learning in text form: prompt examples are the support set, and the next completion is the query prediction. Source: `[1:12:33]` to `[1:17:30]`.
