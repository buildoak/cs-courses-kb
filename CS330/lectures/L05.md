# L05: Optimization-Based Meta-Learning

This lecture swaps the black-box inner learner for gradient descent itself: the meta-learner now searches for an initialization that adapts fast. Source: `[3:54]` to `[4:18]`.

## Summary
- The lecture reframes meta-learning as learning an initialization theta that becomes useful after one or a few inner-loop gradient steps. Source: `[4:08]` to `[7:25]`.
- MAML is the canonical example: inner-loop adaptation happens on the support set, then the outer loop optimizes query performance after that adaptation. Source: `[7:57]` to `[13:21]`, `[39:07]` to `[40:27]`.
- The method is a bi-level optimization problem, so second-order derivatives matter unless you explicitly choose a first-order approximation. Source: `[14:10]` to `[15:32]`, `[46:23]` to `[1:08:24]`.
- The lecture spends real time on stability hacks: larger or learned inner learning rates, partial parameter updates, and explicit first-order variants. Source: `[19:07]` to `[20:36]`, `[1:02:19]` to `[1:09:25]`.
- The land-cover case study shows why this matters: region shift makes a good initialization better than vanilla pretraining when the target domain differs. Source: `[1:11:47]` to `[1:16:18]`.

## Key Concepts
- **Inner loop.** Task-specific gradient descent produces phi_i from theta using the task's support set. Source: `[9:38]` to `[11:24]`.
- **Outer loop.** The meta-update differentiates through the inner loop and improves theta using query loss. Source: `[11:07]` to `[13:21]`, `[39:07]` to `[40:27]`.
- **Support/query split.** The support set is for adaptation; the query set measures whether the adaptation actually worked. Source: `[6:13]` to `[7:13]`, `[16:25]` to `[16:35]`.
- **Higher-order gradients.** The outer gradient flows through the inner update, which is why the method is computationally heavier than ordinary fine-tuning. Source: `[40:27]` to `[46:26]`, `[1:11:03]` to `[1:11:13]`.
- **Inner learning rate alpha.** Alpha is a real meta-parameter here; the lecture notes that it is often much larger than a standard SGD step size. Source: `[19:07]` to `[19:27]`.
- **First-order approximations.** The lecture explains the common trick of treating the inner gradient path as identity or otherwise dropping second-order terms. Source: `[1:05:01]` to `[1:06:46]`.
- **BOIL / representation change.** One variant only updates part of the network in the inner loop, which can be better when the feature representation itself needs to move. Source: `[1:09:02]` to `[1:09:25]`.

## Non-Obvious Insights
- MAML is not "a classifier with more steps." It is an initialization search problem with a very specific geometry: find parameters that are already close to many task-specific optima. Source: `[17:24]` to `[19:27]`.
- The inner-loop learning rate is intentionally aggressive. If it were ordinary SGD scale, the model would not travel far enough in task space. Source: `[19:07]` to `[19:27]`.
- The core advantage over black-box meta-learning is inductive bias. The model starts with gradient descent baked in, so it extrapolates better on shifted tasks. Source: `[1:10:04]` to `[1:10:54]`.
- The cost is not just compute. Differentiating through long inner loops is also a memory problem. Source: `[46:23]` to `[48:56]`, `[1:11:03]` to `[1:11:13]`.
- A task can have its own loss function; the inner and outer loop just need to use the right task-specific objective consistently. Source: `[15:06]` to `[15:27]`.
- The optimum picture is misleading if you read it literally. The real target is a region of parameter space that is easy to adapt from, not one magical point. Source: `[18:21]` to `[19:27]`.
