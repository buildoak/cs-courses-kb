# CS330 L18: Percy Liang Guest Lecture

## Summary
- The lecture treats in-context learning as a real learning mechanism, not just fancy prompt completion.
- On synthetic function classes, transformers can approximate least squares on dense linear tasks and lasso on sparse ones.
- The behavior is not exact, and the failure modes matter: label noise triggers double-descent-like behavior.
- A Bayesian view makes the prompt into a posterior predictive problem, with the task prior carried by the model.
- Random labels and label shuffling are the useful stress tests. They show that the model is inferring task structure, not just memorizing examples.

## Key Concepts
- **In-context learning.** Learning from examples inside the prompt without parameter updates.
- **Conditioning.** The Bayesian alternative to gradient-based training in this setting.
- **Function class.** A family of tasks used to test whether the model can infer the underlying rule.
- **Least squares.** The dense linear baseline the transformer is compared against.
- **Lasso.** The sparse baseline for sparse linear functions.
- **Double descent.** The noise-sensitive failure mode that appears under label noise.
- **Posterior predictive distribution.** The Bayesian object that the model seems to approximate.
- **Task prior.** The model's implicit prior over what the prompt is asking for.
- **Variable binding.** Reinterpreting labels based on prompt context.
- **Instruction tuning.** A practical way to make in-context behavior more reliable.
- **Multimodal ICL.** The same general idea extended beyond text.

## Non-Obvious Insights
- The transformer is not literally running least squares, but its behavior can still look algorithmically close to it.
- The prompt examples are doing two jobs at once: they reveal the task and they reveal the format.
- Order is not the point in the IID synthetic setup. The content of the examples is.
- Label noise is a good probe because it exposes when the model is averaging evidence versus just pattern-matching.
- Random labels are not as absurd as they sound. The prompt still carries enough structure for the model to infer the task.
- The talk quietly shifts the research question. It is no longer "can the model answer?" It is "what prior and conditioning machinery makes the answer possible?"
- Instruction tuning can make the system more practical, but the scientific surprise is that the raw pretrained model already has some of the machinery.
