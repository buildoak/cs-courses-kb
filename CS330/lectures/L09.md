# CS330 L09: Task Construction for Meta-Learning

## Summary
- Meta-learning can fail by memorization when the task family is constructed so a single function can solve everything.
- If task labels are not mutually exclusive, the learner can ignore the support set and stash the task in `theta`.
- Label shuffling and other forms of task exclusivity force the model to use the episodic data.
- The lecture then flips the problem around: build tasks automatically from unlabeled data.
- Two main recipes appear: cluster-based image tasks and masked-word text tasks, often with semi-supervised hybrids.

## Key Concepts
- **Support / query split.** The support set defines the task; the query set measures generalization.
- **Mutual exclusivity.** If tasks are labeled inconsistently, one global classifier cannot solve them all.
- **Memorization.** The model stores task identity in parameters instead of learning to adapt from data.
- **Information flow.** The question is whether prediction depends on `theta` or on the task data.
- **CACTUs.** Clustering-based construction of pseudo-tasks for unsupervised meta-learning.
- **Augmentation-based tasks.** Build episodic tasks from image transformations that preserve class identity.
- **SMLMT.** Masked-word tasks for text that turn unlabeled language into few-shot episodes.
- **Semi-supervised hybrids.** Mix unlabeled pseudo-tasks with labeled tasks when both are available.

## Non-Obvious Insights
- The real failure mode is not optimization. It is task design that lets the learner bypass adaptation.
- Validation tasks can reveal memorization, but they do not automatically fix it. The data construction has to change.
- Unsupervised task construction is only as good as the representation you cluster or augment from.
- The lecture treats task construction as an information-routing problem: make the support set informative and the shortcut less attractive.
- For text, the best results often come from hybrids, not from pure unsupervised task generation.
