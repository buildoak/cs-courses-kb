# CS330 L15: Lifelong Learning

## Summary
- Lifelong learning is a messy umbrella term for sequential, continual, incremental, and streaming learning.
- The lecture uses a simple online setup: observe an input, predict, observe the label, repeat.
- Regret is the main theoretical lens, along with forward transfer and backward transfer.
- Follow-the-leader stores everything and retrains, while SGD is cheap but forgets past data.
- GEM adds episodic memory plus gradient constraints so new updates do not damage old tasks.
- The lecture ends by reframing the problem as online meta-learning, where the goal is to learn a good update rule.

## Key Concepts
- **Sequential learning.** Data or tasks arrive over time.
- **Streaming setting.** You may not be allowed to store everything.
- **Regret.** Cumulative loss minus the best hindsight comparator.
- **Forward transfer.** Past tasks help future tasks.
- **Backward transfer.** New tasks help or hurt past tasks.
- **Catastrophic forgetting.** Negative backward transfer from overwriting old knowledge.
- **Follow-the-leader.** Retrain on all seen data at each step.
- **SGD.** Update only on the current example or batch.
- **Episodic memory.** Keep a tiny replay buffer per task.
- **GEM.** Gradient Episodic Memory, the constrained-memory method.
- **Local linearity.** The gradient constraint is only a proxy for the true loss constraint.
- **Online meta-learning.** Learn an update rule that adapts quickly on new tasks.

## Non-Obvious Insights
- Regret only makes clean sense when there is a single comparator that could have handled all tasks.
- Negative backward transfer can come from memory limits, task order, or insufficient model capacity.
- The memory-based constraint in GEM is an approximation, not a guarantee; it assumes local linearity.
- Synthetic lifelong-learning benchmarks can hide the fact that the whole problem may be trivial to store on a laptop.
- Meta-learning is not just for few-shot classification here. It can be used to learn update rules that avoid forgetting.

## Connections
