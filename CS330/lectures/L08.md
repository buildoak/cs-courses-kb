# CS330 L08: Reconstruction-Based Pre-Training for Few-Shot Learning

## Summary
- Reconstruction-based pre-training learns a representation by encoding an input and decoding it back into the original signal.
- Naive autoencoders collapse to identity mappings unless the representation is constrained.
- Low-dimensional bottlenecks help, but often produce hashes of the input rather than reusable features.
- Masked autoencoders fix the task, not just the bottleneck: predict missing pieces from visible pieces.
- The lecture then connects this idea to BERT, autoregressive models, LoRA, and Flamingo.

## Key Concepts
- **Autoencoder.** Encode an input into a latent representation and decode it back.
- **Reconstruction loss.** Measure how close the reconstruction is to the original input.
- **Identity collapse.** If the latent is unconstrained, the model can just copy the input through.
- **Bottleneck.** A constraint on the latent, usually by lowering dimensionality or adding noise.
- **Masked autoencoder.** Hide part of the input and reconstruct the missing part.
- **BERT.** The classic masked-language-model version of the same idea.
- **Autoregressive model.** Predict the next token from a prefix; this is a special case of masking.
- **LoRA.** Fine-tuning as a low-rank update to frozen pre-trained weights.
- **Flamingo.** Multimodal autoregressive pretraining that combines vision and language.

## Non-Obvious Insights
- A smaller latent is not automatically a better representation. You can compress an input into a useless hash.
- Masking changes the task in a way dimensional bottlenecks do not. The encoder must infer missing structure, not just compress pixels.
- Autoregressive transformers buy efficiency by caching prefix representations, but they lose bidirectional context.
- The lecture's LoRA detour is not a tangent. It shows fine-tuning as a low-rank perturbation of the pre-trained operator.
- Masked autoencoders and autoregressive models are the same family. The difference is the mask shape and the attention mask.
