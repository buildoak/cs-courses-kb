# CS330 L13: Domain Adaptation

## Summary
- Domain adaptation is the setting where you want target-domain performance while still training on source labels plus unlabeled target data.
- The lecture treats this as transductive learning: test-domain data is available during training, but usually without labels.
- Importance weighting handles covariate shift by reweighting source examples by the `p_target(x) / p_source(x)` ratio.
- Domain-adversarial training learns features that confuse a domain classifier while still supporting label prediction.
- CycleGAN-style translation maps source examples into the target style, then forces the mapping to be cycle-consistent.

## Key Concepts
- **Source / target domain.** Same task, different input distribution.
- **Transductive learning.** Some target examples are visible during training.
- **Covariate shift.** `p(x)` changes, while `p(y|x)` stays fixed.
- **Importance weighting.** Reweight source examples to match the target distribution.
- **Density ratio.** Estimate `p_target(x) / p_source(x)` instead of two full densities.
- **Support coverage.** The source must cover the target regions you care about.
- **Feature alignment.** Learn a representation where source and target look similar.
- **Gradient reversal.** Push the encoder to fool the domain classifier.
- **Cycle consistency.** Mapping source -> target -> source should recover the original sample.

## Non-Obvious Insights
- Importance weighting is not a general fix. It only works when the source support covers the target support.
- The classifier trick for the density ratio is practical because high-dimensional density estimation is brittle.
- Domain-adversarial training is iterative, not one-shot. A frozen domain classifier can be gamed by bad features.
- Feature alignment is population-level alignment, not per-example matching.
- CycleGAN works because the mapping is constrained twice: by adversarial realism and by the cycle loss.
