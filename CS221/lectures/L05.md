# L05: Reinforcement Learning

## Summary
- Reinforcement learning keeps the MDP skeleton but removes the model: transitions and rewards are unknown, so the agent has to learn from experience.
- Model-based Monte Carlo estimates the MDP from sampled episodes, then solves the estimated MDP with planning.
- Model-free Monte Carlo estimates `Q_pi` directly from returns; SARSA bootstraps from the next action-value; Q-learning bootstraps toward the optimal action-value.
- Exploration is the central practical problem. If you never visit a branch, you can miss the reward hiding there.
- Function approximation turns tabular Q-learning into a feature-based update, which is the bridge to generalization.

## Key Concepts
- **Agent / environment.** The agent chooses actions; the environment returns rewards and next states.
- **Episode.** A trajectory of `(state, action, reward, state, ...)` tuples until termination.
- **Utility / return.** The discounted sum of rewards from a point in an episode.
- **Model-based Monte Carlo.** Estimate transition counts and rewards from data, then run planning on the estimated MDP.
- **Model-free Monte Carlo.** Estimate `Q_pi` by averaging observed returns for each `(s, a)` pair.
- **SARSA.** On-policy bootstrapping: update toward `r + gamma * Q_pi(s', a')`.
- **Q-learning.** Off-policy bootstrapping: update toward `r + gamma * max_a Q(s', a)`.
- **On-policy vs off-policy.** On-policy learns the value of the policy it follows; off-policy learns the optimal value while behaving differently.
- **Exploration policy.** The policy used to gather data, which may differ from the policy being learned.
- **Epsilon-greedy.** Mostly exploit the best-known action, but occasionally explore randomly.
- **Function approximation.** Replace a lookup table with weights and features, so similar states can share information.
- **SGD-style update.** The Q-learning update becomes a familiar residual-times-feature-vector step.

## Non-Obvious Insights
- "Model-free" in RL does not mean "no model whatsoever." It means no explicit MDP model. The algorithm still estimates value functions.
- Model-free Monte Carlo has high variance because it waits for full returns; SARSA reduces variance by bootstrapping, but pays with bias.
- Q-learning can learn the optimal policy while following a random or exploratory behavior policy. That is the whole point of off-policy learning.
- Exploration is not optional. A greedy policy can lock itself into a locally good branch and never discover the better one.
- The function-approximation section is basically linear regression in RL clothing: `Q_opt(s, a) = w dot phi(s, a)`.
- Deep RL in the lecture is not a different idea. It is the same `Q_opt` target with a neural network instead of a linear feature model.
