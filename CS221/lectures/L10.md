# L10: Feature Templates

## Summary
- Feature templates are a way to describe families of features, not just one-off indicators. The learner gets the instantiated features, not the template itself. Source: `[5:59]` to `[7:06]`.
- The lecture starts with email validity as the toy task: length, alphanumeric fraction, `@`, `.com`, and `.org` all become votes in a linear classifier. Source: `[1:47]` to `[5:19]`.
- A template is just a blanked-out rule like "last three characters equals ___" or "length greater than ___". Instantiating the blank turns one pattern into many concrete features. Source: `[6:09]` to `[7:43]`.
- Feature templates also work for structured inputs like images plus metadata, where you can ask about a pixel at a row, column, and color channel, or about a lat/long region. Source: `[7:44]` to `[9:09]`.
- Sparse feature vectors are the practical punchline. If most entries are zero, a dictionary is a better representation than an array. Source: `[9:10]` to `[11:43]`.

## Key Concepts
- **Hypothesis class.** The set of predictors the learner is allowed to choose from. Source: `[0:14]` to `[1:15]`.
- **Feature extractor `phi`.** The function that turns raw input into a feature vector, often using prior knowledge. Source: `[1:53]` to `[3:26]`.
- **Feature name/value pairs.** A sparse-friendly way to describe features before they are flattened into a vector. Source: `[2:37]` to `[3:26]`.
- **Vote interpretation.** In a linear classifier, each feature votes for or against the positive class depending on the sign of its weight. Source: `[4:37]` to `[5:23]`.
- **Feature template.** A reusable description with a blank that can be instantiated many times. Source: `[5:59]` to `[7:06]`.
- **Sparse vector.** A vector with many zero entries. Source: `[9:10]` to `[9:44]`.
- **Dictionary representation.** Store only non-zero feature names and values instead of a dense array. Source: `[9:33]` to `[10:25]`.
- **Lazy instantiation.** Sparse features can be created as data arrives instead of being predefined all at once. Source: `[10:31]` to `[10:46]`.

## Non-Obvious Insights
- The template is the important object, not any single instantiated feature. That is the real compression trick. Source: `[6:19]` to `[7:06]`.
- Feature design is about saying which properties might matter, not deciding the final answer up front. The learning algorithm still sorts out the weights from data. Source: `[2:15]` to `[2:32]`.
- Sparse representation is part of the model design, not just an implementation detail. If the feature space explodes, storage strategy becomes part of the solution. Source: `[9:10]` to `[10:25]`.
- Dense arrays are still useful when the features are mostly non-zero, especially when matrix computation is the bottleneck. Source: `[10:47]` to `[11:10]`.
- The lecture is quietly showing the same move as feature engineering at scale: define a pattern class, then let data choose which instances matter. Source: `[5:51]` to `[7:06]`.

## Connections
- `L09` showed how changing `phi(x)` changes what a linear model can express; this lecture shows how to scale that idea without hand-listing every feature.
- `L11` replaces manual templates with learned intermediate features.
- `L12` gives the gradient machinery needed to train whatever feature representation you choose.
- The email and image examples both make the same point: the best feature is often a family of related tests, not a single handcrafted scalar.
