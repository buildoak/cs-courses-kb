## Surprising Connections
- A* and value iteration both come from the same move: relax the hard problem until the remaining cost is computable, then reuse that structure to prune the search. [L17](/Users/otonashi/thinking/building/cs-courses-kb/CS221/lectures/L17.md), [L19](/Users/otonashi/thinking/building/cs-courses-kb/CS221/lectures/L19.md)
- Q-learning is value iteration with experience instead of a model, and forward-backward is dynamic programming on a compressed lattice rather than a brute-force enumeration of paths. [L15](/Users/otonashi/thinking/building/cs-courses-kb/CS221/lectures/L15.md), [L20](/Users/otonashi/thinking/building/cs-courses-kb/CS221/lectures/L20.md)
- Inference: k-means, value iteration, and policy iteration all share the same outer loop shape. Fix one side of the problem, optimize the other, and repeat until the representation settles. [L04](/Users/otonashi/thinking/building/cs-courses-kb/CS221/lectures/L04.md), [L16](/Users/otonashi/thinking/building/cs-courses-kb/CS221/lectures/L16.md), [L19](/Users/otonashi/thinking/building/cs-courses-kb/CS221/lectures/L19.md)
- Backprop is the computation-graph version of the same "reuse the local rule" idea that shows up in probabilistic inference. The graph changes, but the work is still local message passing. [L12](/Users/otonashi/thinking/building/cs-courses-kb/CS221/lectures/L12.md), [L15](/Users/otonashi/thinking/building/cs-courses-kb/CS221/lectures/L15.md)
- AI amplification and group DRO are the same moral warning at different scales. If the objective is average loss or raw capability, the worst-off slice can get crushed while the headline metric improves. [L03](/Users/otonashi/thinking/building/cs-courses-kb/CS221/lectures/L03.md), [L08](/Users/otonashi/thinking/building/cs-courses-kb/CS221/lectures/L08.md)

## Under-Appreciated Ideas
- State design is the main algorithmic lever. Add memory bits and search can collapse from quadratic-ish to linear-ish; choose the wrong state and the whole problem stays expensive. [L17](/Users/otonashi/thinking/building/cs-courses-kb/CS221/lectures/L17.md)
- Relaxation is the heuristic factory. Remove walls, remove constraints, or split the problem into independent pieces until the answer becomes closed form, then reuse that answer as `h(s)`. [L18](/Users/otonashi/thinking/building/cs-courses-kb/CS221/lectures/L18.md), [L19](/Users/otonashi/thinking/building/cs-courses-kb/CS221/lectures/L19.md)
- K-means is not just clustering; it is representation building. The centroids become features for later supervised work, so the unsupervised step is part of the model pipeline, not a side quest. [L16](/Users/otonashi/thinking/building/cs-courses-kb/CS221/lectures/L16.md)
- Feature templates and sparse dictionaries are infrastructure, not implementation trivia. They are how CS221 scales hand-designed features without pretending the feature space is small. [L10](/Users/otonashi/thinking/building/cs-courses-kb/CS221/lectures/L10.md)
- Backprop is generic differentiation infrastructure, not a neural-net special case. Once the computation graph is explicit, the same local rules train linear models, two-layer nets, and deeper stacks. [L11](/Users/otonashi/thinking/building/cs-courses-kb/CS221/lectures/L11.md), [L12](/Users/otonashi/thinking/building/cs-courses-kb/CS221/lectures/L12.md), [L10](/Users/otonashi/thinking/building/cs-courses-kb/CS221/lectures/L10.md)
- `gamma` is a stabilization knob as much as a modeling choice. It keeps cyclic MDPs numerically sane and makes the recurrence behave. [L04](/Users/otonashi/thinking/building/cs-courses-kb/CS221/lectures/L04.md), [L19](/Users/otonashi/thinking/building/cs-courses-kb/CS221/lectures/L19.md)
- Exploration is not optional in RL. Without it, the agent can miss reward in states it never visits, and the whole policy becomes a local story. [L05](/Users/otonashi/thinking/building/cs-courses-kb/CS221/lectures/L05.md), [L20](/Users/otonashi/thinking/building/cs-courses-kb/CS221/lectures/L20.md)
